Summary

当前角色页已经使用正确角色头像与 Banner 资源，但页面结构、卡片比例和主视觉心形都明显简化，离设计稿仍有较大差距。

---------------------------------------

P0（必须修复）

- 角色卡片结构与设计稿差距过大。
原因：设计稿的角色卡片更高、更宽、头像更大、描述文本更多；当前实现压缩成了两张较短卡片。
建议修改方式：重做角色卡片高度、头像尺寸、文本区域宽度、选择态位置和整体留白。
涉及组件：角色卡片。
涉及文件：`web/src/pages/CharacterPage.tsx`

- 顶部 Hero 心形被错误地改成了简化 SVG。
原因：设计稿是更精致、更通透的玻璃心形主视觉；当前 SVG 造型、渐变和高光都不对。
建议修改方式：按设计稿还原主视觉心形的比例和光感，不要继续沿用当前简化矢量。
涉及组件：Hero 主视觉。
涉及文件：`web/src/pages/CharacterPage.tsx`

- 底部 CTA 承载方式不对。
原因：设计稿中按钮是漂浮在背景之上的主 CTA；当前底部有一整块实体色容器，破坏了轻盈感。
建议修改方式：移除当前底部实体条，改为设计稿式浮动按钮区。
涉及组件：底部 CTA。
涉及文件：`web/src/pages/CharacterPage.tsx`

---------------------------------------

P1（建议修复）

- 页面标题与返回区的视觉张力偏弱，未形成设计稿顶部引导感。
涉及组件：顶部导航。
涉及文件：`web/src/pages/CharacterPage.tsx`

- 选中态粉色圆形勾选和未选中态描边按钮都偏小。
涉及组件：选择态控件。
涉及文件：`web/src/pages/CharacterPage.tsx`

---------------------------------------

P2（可优化）

- 当前选择成功 Toast 没有独立 QA 预览入口，不利于后续统一验收反馈层。
涉及组件：Toast。
涉及文件：`web/src/pages/CharacterPage.tsx`、`web/src/components/ui/Toast.tsx`

---------------------------------------

Assets

- 资源引用正确：`亮色背景图.png`、`暗色背景图.png`、`background_character_hero.webp`、两张角色头像资源。
- 问题不在资源缺失，而在主视觉被简化复刻。
