Summary

当前实现使用了正确资源，但整体版式与设计稿差距很大，更像“简化轮播页”，不是设计稿中的完整 onboarding 视觉结构。

---------------------------------------

P0（必须修复）

- 当前页面结构与设计稿不一致。
原因：设计稿是完整的 onboarding 卡片式构图，包含更强的图像占比、文案层级、分页点和按钮关系；当前实现只是“上方图片 + 下方文案 + CTA”。
建议修改方式：重做 `OnboardingPage` 的整体版式，按设计稿重建首屏、第二屏、第三屏的垂直比例和底部操作区。
涉及组件：页面整体布局、分页点、主按钮。
涉及文件：`web/src/pages/OnboardingPage.tsx`

- 第三步缺少设计稿中的兑换码捷径。
原因：设计稿第三屏存在“我有兑换码”次级路径，当前页面只有“开始体验”。
建议修改方式：在最后一步底部加入次级激活入口，并与 `Redeem` 页面打通。
涉及组件：次级 CTA。
涉及文件：`web/src/pages/OnboardingPage.tsx`

- 插图显示尺寸过小，弱化了设计稿的主视觉权重。
原因：当前图片被限制在 `max-w-[300px]`，导致大面积留白与设计稿不一致。
建议修改方式：按设计稿重新定义插图区域高度和最大宽度，不要使用当前通用居中缩放逻辑。
涉及组件：图片容器。
涉及文件：`web/src/pages/OnboardingPage.tsx`

---------------------------------------

P1（建议修复）

- 标题与描述文案层级偏弱，字号、行距、底部留白都明显小于设计稿。
涉及组件：Typography。
涉及文件：`web/src/pages/OnboardingPage.tsx`、`web/src/components/ui/Button.tsx`

- 当前返回按钮只在第 2、3 步出现，视觉位置正确但与整体页面版式风格割裂。
涉及组件：顶部导航。
涉及文件：`web/src/pages/OnboardingPage.tsx`

涉及组件：页面转场。
涉及文件：`web/src/pages/OnboardingPage.tsx`、`web/src/index.css`
