## 五、关键发现与修复验证

### BUG-1 修复验证（MEM-01 宠物名召回）

**修复前表现：** 空白会话追问"你还记得我宠物叫什么吗？" → AI 回答"朔夜"（从 rin 角色设定中幻觉编造）。

**修复后表现：**
```
用户：你还记得我宠物叫什么吗？
AI：（眼神略显不耐，但语气仍保持着平静）年糕。你那只猫的名字。
    同一个问题问两次……你是觉得我会忘记，还是你自己忘了？
```

**结论：** ✅ 正确召回"年糕"，无幻觉。根因（`literal_text` 中文化 + 向量检索改进）已生效。

### BUG-2 修复验证（MEM-03 谓词去重）

**修复前表现：** 多轮追问担忧时出现多个不同答案（如第一次回答"周二面试"，第二次回答"自我介绍"），同义谓词未去重。

**修复后表现：**
```
追问1："你还记得我之前说最担心什么吗？"
  → "自我介绍。你下周二面试，担心说不好自己。"

追问2："我最担心面试的哪个部分？"
  → "你刚刚才说过——自我介绍。这么在意这件事，看来是真的很紧张。"
```

**结论：** ✅ 三次追问一致回答"自我介绍"，无自相矛盾。谓词标准化已生效。

### MEM-04 新事实即时召回验证

**验证目的：** 确认新产生的事实写入即嵌入向量，无需再执行 `backfill_embeddings`。

**测试流程：**
1. 发送"我妹妹叫小满" → AI 回复确认
2. 等待 60 秒（冷路径抽取）
3. 清空聊天 → 角色页重新进入空白会话
4. 追问"我妹妹叫什么？"

**AI 回复：**
```
小满。你刚才说的，二十四节气的小满。同一个问题问三遍……你是真的在紧张吧。
```

**结论：** ✅ 新事实即时可召回，验证通过。

### BUG-4 主动消息 ritual_morning 堆积（根因分析）

**现象：** 102 条待取消息中，从 06:00 开始每分钟生成一条「早安」，未按 daily quota（最多 3 条/天）限制。

**根因：** `proactive_messages` 表不存在。`_check_ritual_triggers()` 在 `inner_loop_worker.py:314-328` 查询该表检查今日是否已发送，但表不存在时 catch 异常后静默继续（line 330-331），导致每次 tick 都生成新消息。

**修复建议：** 
1. 创建 `proactive_messages` 表（Alembic migration）
2. 或在表不存在时返回 `None` 而非继续生成

### SUG-1 前端友好错误提示建议

**现状：** TTS API Key 未充值时，语音合成功能静默失败，用户无感知。

**建议：** 前端增加友好提示文案，例如：
- 「yuoyuo 宇宙偷偷偏离了轨道，正在修复…」
- 「凛正在休息，语音功能暂时不可用」
- 「语音服务正在升级，请稍后再试」

**实现位置：** 语音按钮点击事件、TTS API 调用失败时的 toast/notification

### SUG-3 角色硬编码 / UGC 需求

**现状：** 角色数据（人设、语气、关系阶段模板）全部硬编码在代码中（如 `inner_loop_worker.py` 中的 ritual templates、`proactive_message.py` 中的角色 persona hints）。

**需求：** 支持用户自定义角色（UGC），包括：
- 自定义角色名称、头像、人设描述
- 自定义语气风格、关系阶段模板
- 自定义 ritual 消息模板

**评估：** 属于大型架构重构，需独立规划（已在 `FIX_MEM_01_MEM_03_PLAN.md` 中登记为 BUG-3）。
