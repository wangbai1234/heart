## 发现的问题

### 问题 1：测试 2 - 超过 40 条窗口后早期信息丢失

**现象**：
- AI 在对话过程中多次正确使用了"年糕"这个名字
- 但在最后追问时，AI 回答了错误的名字"六子"

**原因分析**：
- L1 工作记忆窗口限制（最近 10-30 turns）
- L3 语义记忆中有正确的"年糕"记录（confidence: 1.0）
- 但记忆召回逻辑可能未正确触发

---

### 问题 2：L4 层记忆数据异常

**现象**：
```json
{
  "key": "name",
  "value": "什么吗",  // ⚠️ 数据异常
  "disclosure_context": "我叫什么吗"
}
```

**问题**：
- L4 层记忆 `value` 字段为"什么吗"，这是错误数据

**原因（已代码走查修正）**：
- ⚠️ **原判断"fast_encoder_identity_detection 误识别"有误**：该身份识别逻辑已废弃，
  `backend/heart/ss02_memory/encoder/fast.py:104` 恒返回 `[]`，不参与 L4 写入。
- 真正原因：**LLM 抽取器把疑问句"我叫什么吗"错标成了 disclosure**（应标 `kind=question`），
  产出 `value="什么吗"` 的 L3 事实，绕过了 Resolver 已有的 question 拦截
  （`backend/heart/ss02_memory/extractor/resolver.py:109`），再经晋升进 L4。
- 修复见 `docs/MEMORY_FIX_PLAN.md` P0-3（抽取 prompt + Resolver 防御式校验 + 脏数据清理）。

**影响**：
- 身份记忆不准确
- 可能影响角色对用户的称呼

---

### 问题 3：L2 层情感分析异常

**现象**：
- 所有记忆的情感峰值都是 `arousal:0.5, valence:0`
- 只有"我讨厌香菜"的 `valence:-1`（负面）
- 只有"好的好的"的 `valence:1`（正面）

**问题**：
- 情感分析没有正确区分不同消息的情感
- 大部分记忆没有正确分析情感

**影响**：
- 情感状态不准确
- 无法根据情感调整回复风格

---

### 问题 4：L2 层重要性评分异常

**现象**：
- 大部分记忆的重要性都是 0.3
- 只有"好的好的"和"我讨厌香菜"是 1.0

**问题**：
- 重要性评分没有根据内容差异化
- 普通消息和重要信息评分相同

**影响**：
- 记忆优先级不正确
- 重要信息可能被忽略

---

### 问题 5：L2 层记忆从未被召回

**现象**：
- 所有记忆的 `recall_count` 都是 0
- 所有记忆的 `last_recalled_at` 都是 null

**问题**：
- 记忆从未被召回过
- 记忆系统可能未正常工作

**影响**：
- 记忆无法被使用
- 角色无法回忆之前的对话

---

### 问题 6：L2 层场景上下文缺失

**现象**：
- 所有记忆的 `scene_context` 都是 null
- 没有记录对话场景

**问题**：
- 无法还原对话场景
- 缺少上下文信息

**影响**：
- 记忆召回时缺乏场景信息
- 无法提供更自然的回复

---

### 问题 7：L2/L3 层记忆不一致

**现象**：

| 记忆内容 | L2 层状态 | L3 层状态 |
|----------|-----------|-----------|
| 我养了一只猫叫年糕 | ✅ 有记录 | ✅ 有记录 |
| 我在苏州工作 | ✅ 有记录 | ✅ 有记录 |
| 我讨厌香菜 | ✅ 有记录 | ✅ 有记录 |
| 我生日是3月14日 | ✅ 有记录 | ⚠️ 未找到 |
| 我住在上海 | ⚠️ 未找到 | ✅ 有记录 |
| 我搬到深圳了 | ⚠️ 未找到 | ✅ 有记录 |

**问题**：
- L2 和 L3 层记忆不同步
- 部分记忆只存在于一层

**影响**：
- 记忆一致性差
- 可能导致召回失败

---

### 问题 8：情感状态异常

**现象**：
- 关系阶段：ACQUAINTANCE（熟人）
- 特殊状态：**drifting**（漂移中）
- 情感事件表为空

**问题**：
- drifting 状态可能导致情感表达不稳定
- 没有情感事件记录，无法追踪情感变化

**影响**：
- 情感状态不准确
- 无法提供一致的情感体验
