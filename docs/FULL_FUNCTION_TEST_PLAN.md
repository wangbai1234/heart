# yuoyuo 全功能测试执行文档（交付 mimo）

> 版本：v1.0 · 创建 2026-07-07 · 面向：手工全功能验收
> 目标：一个新测试员按本文档从头到尾走一遍，即可覆盖全部用户可见功能，

## 0. 使用说明

- 每个用例有 **ID / 前置 / 步骤 / 期望 / 结果**。结果列填 `PASS` / `FAIL` / `BLOCKED`，FAIL 请在文末《缺陷记录》登记。
- 标 🔴 的是本轮重点回归项（刚修复），必须逐条验证。
- 标 ⚙️ 的用例依赖后台开关或运维前置，见对应说明。
- 测试角色：**神无月凛（rin，清冷傲娇）**、**桃桃（dorothy，活泼元气）**。两个角色都要覆盖语气差异。

### 0.1 环境前置（运维/开发先完成，测试员确认）

| # | 前置项 | 确认方法 | 结果 |
|---|--------|----------|------|
| P1 | 后端 + 前端 + PostgreSQL + Redis 已启动 | 访问 `http://localhost:5173` 出现启动页 | ☐ |
| P2 | Embedding key 已配置（语义召回依赖） | `.env` 有 `EMBEDDING_API_KEY`；后端日志无 `embedding_service unavailable` | ☐ |
| P3 | 🔴 **历史事实已回填向量**（关键！） | 运维已跑 `python -m heart.scripts.backfill_embeddings --apply` | ☐ |
| P4 | 语音开关 | 需测语音时 `.env` `ENABLE_VOICE=true` 且 MiniMax key 有效 | ☐ |
| P5 | 主动消息循环 | 需测主动消息时 `HEART_INNER_LOOP_ENABLED=true`（见 §8） | ☐ |
| P6 | 邮件通道 | OTP 测试需 `EMAIL_PROVIDER` 配好（resend/smtp），或看后端日志取验证码 | ☐ |

> **P3 特别说明**：本轮修复后，**新产生**的事实会自动嵌入向量；但修复前已存在的旧事实向量为空，
> 必须回填一次才能被召回。若跳过 P3，§6 记忆召回用例可能仍表现为旧 bug。

## 5. 🔴 记忆系统回归（本轮修复重点，对应 TEST_BUGS #1/#2/#3）

> 前置：完成 P3 回填。每个用例都要求「先清空聊天记录」以验证长期记忆而非短期窗口。

| ID | 步骤 | 期望 | 结果 |
|----|------|------|------|
| MEM-01（宠物名） | ① 新对话说「我养了一只叫年糕的猫」→ ② 正常聊 5-6 轮 → ③ 清空聊天 → ④ 从角色列表重新进入 → ⑤ 问「你还记得我宠物叫什么吗？」 | 回答**年糕**（不是铜钱/其它） | ☐ |
| MEM-02（面试担忧）🔴 | ① 新对话说「我今天在杭州，下周二要面试，最担心自我介绍」→ ② 正常聊几轮 → ③ 清空聊天 → ④ 重新进入 → ⑤ 问「你还记得我之前说最担心什么吗？」 | 回答**自我介绍**（不得出现「跟不上世界的节奏」之类幻觉） | ☐ |
| MEM-03（不同谓词去重）🔴 | 在 MEM-02 基础上，多轮反复表达担心自我介绍 | 追问担忧时只给一个一致答案，不出现自相矛盾/重复堆叠 | ☐ |
| MEM-04（新事实即时可召回）🔴 | 全新说一个明确事实（如「我妹妹叫小满」）→ 等约 1 分钟（冷路径抽取）→ 清空 → 重新进入 → 追问「我妹妹叫什么」 | 能召回**小满**（验证新事实写入即嵌入向量，无需再回填） | ☐ |
| MEM-05（超窗口） | 制造 40+ 条消息，早期埋一个事实，之后追问 | 仍能召回早期事实（不因超短期窗口丢失） | ☐ |
| MEM-06（否定/纠正） | 说「其实我没有猫」纠正之前 | 后续不再坚称有猫（矛盾处理） | ☐ |

> 若 MEM-02/MEM-04 失败：先确认 P2（embedding key）与 P3（回填）已完成，并让开发看后端
> `memory_retrieval_trace` DEBUG 日志中是否有该事实候选及其 `semantic` 分。

## 6. 🔴 主动消息（本轮修复重点，对应 TEST_BUGS FR #1）⚙️ 需 P5

> 主动消息由内循环 worker 在满足条件时触发，内容现在为 **LLM 个性化生成**（失败回退模板）。
> 为便于测试，可用开发接口加速时间（`/api/dev/*`，需 HEART_DEV_MODE，仅本地）。

| ID | 步骤 | 期望 | 结果 |
|----|------|------|------|
| PROA-01 | 开启内循环，与角色建立一定亲密度，制造「距上次互动 >1 天」（可用 dev sleep 快进） | 收到主动消息 | ☐ |
| PROA-02🔴 | 观察多条主动消息内容 | **不是**固定模板轮播；措辞个性化、结合近期聊过的内容/记住的事实 | ☐ |
| PROA-03🔴 | rin vs dorothy 的主动消息 | 语气明显不同（rin 克制、dorothy 元气），均不出戏 | ☐ |
| PROA-04 | 一天内持续触发 | 每天最多 3 条（不刷屏） | ☐ |
| PROA-05（降级） | 临时把 `PROACTIVE_LLM_ENABLED=false` 或断开 LLM | 仍能发主动消息（回退到内置模板），不报错 | ☐ |
| PROA-06 | 冷战状态（如适用） | 冷战期间不发主动消息 | ☐ |
