## 🔴 BUG 列表

### BUG-1（P1）：主动消息大量重复 — dedup 失效

**现象**：所有角色（rin/dorothy/魅魔/月月）的聊天历史中，主动消息严重重复：
- rin: "下雨了。记得带伞。" x5+，"今天有点累…" x4，"今天翻到…" x4
- dorothy: "外面的太阳好大耶！" x2
- 月月: "月月想着你，今天过得怎么样？" x9
- 魅魔: "魅魔一直在想你。" x2

**影响**：聊天界面被垃圾消息淹没，用户体验极差。

**建议排查**：
1. `proactive_messages` 表 dedup 逻辑（可能是 `proactive_generator` 的 `dedup_window` 太短或失效）
2. 定时器重复触发（`HEART_INNER_LOOP_ENABLED=true` 时，ritual dedup 是否真的在查 DB）
3. 前端消息合并逻辑（是否有 client-side dedup）

### BUG-2（P2）：未配音色 UGC 角色无"暂未配置音色"提示

**现象**：对未配置音色的 UGC 角色（魅魔）开启语音开关后发消息，没有看到"该角色暂未配置音色"的文字提示。

**期望**：PR #135 A-4 应在 voice switch ON + 无 voice_id 时显示明确提示。

**可能原因**：toast 闪烁太快被错过，或后端直接返回文本回复（跳过了 voice 生成）。

### BUG-3（P2）：桌面端"删除"按钮行为异常

**现象**：聊天列表的"删除"按钮点击后跳转到聊天页，而非弹出确认对话框。

**期望**：点击"删除"应弹出"确认删除对话？"确认框。

**可能原因**：桌面端的侧滑删除 UI 未适配，"删除"按钮的 click handler 绑定了导航而非确认弹窗。

### BUG-4（P2）：「」拆分在 Windows 半角括号下失败

**现象**：LLM 输出含 `「」` 的文本时，在 Windows 上可能出现括号内句号被错误拆分到独立气泡。

**根因**：`message_splitter.py:103` 的 corner_quote_depth 只统计标准全角 `「」` (U+300C/U+300D)，不统计 Windows IME 常用的半角 `｢｣` (U+FF62/U+FF63)。当 LLM 输出半角括号时，depth 不更新，`。` 在 depth=0 被当作句末终止符触发拆分。

**复现**：`｢今晚别走。我们聊聊。｣` → 拆成两个气泡 `｢今晚别走。` + `我们聊聊。｣`（应保持一个原子气泡）。

**修复方向**：`_split_by_terminators` 中 `part.count("「")` 和 `part.count("」")` 需同时计入半角变体 `｢｣`。

### BUG-5（P1）：历史消息丢失 `kind` 字段 — action 气泡始终渲染为普通文本

**现象**：含 `（动作）` 的 action 消息在实时流中正确渲染为灰色药丸气泡，但从历史加载时渲染为普通文本气泡（无斜体、无灰色背景）。

**根因**：`routes_chat_ws.py:711/723` 的 `get_chat_history` SQL SELECT 缺少 `kind` 列，前端 `loadHistory()` 收到的消息始终 `kind=undefined`，fallback 为 `kind='text'`。

**证据**：DB 中 `chat_messages` 表有 `kind` 列（varchar(16)），rin 有 10 条 `kind='action'` 记录，但 API 返回的 JSON 不含此字段。

**修复方向**：SQL SELECT 加上 `kind`，前端 `loadHistory()` 解析 `kind` 字段。

### BUG-6（P1）：主动消息无每日限额、无跨角色限频、无时间段感知

**现象**：
- 2026-07-12 单日：rin 4条、月月 4条、char_e4cb6142 4条（均超 3 条/日限额）
- 03:16:17 UTC 同一秒内 8 个角色同时发送 ritual_idle 消息
- rin 连续 2 条间隔仅 30 秒（01:49:19 → 01:49:49 UTC）
- 跨角色无门控：rin(03:32) → 哈哈哈(03:35) → 魔女(03:37) → 夜阑(03:39)，7 分钟内 4 个角色各发一条

**根因**：`service.py` 的 `count_all_today` 返回值未被正确强制执行；`proactive_repo.py` 的 dedup 仅按 `trigger_type` 单独去重，无跨字符限频窗口；`activity_generator.py` 无时间段分类逻辑（凌晨/早晨/午后/夜晚应产生不同主题内容）。

**影响**：用户凌晨收到大量无关时间主题的主动消息，且无任何频率限制。
