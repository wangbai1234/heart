## 建议

1. **【P0】UGC 角色创建 API 500 错误**：POST /api/characters 返回 500，导致无法创建新角色，需立即排查修复
2. **【P0】GET /api/characters 返回 500**：虽然前端角色列表仍可正常显示，但 API 层面存在内部错误，需排查
3. **AUTH-04 前端提示优化**：在第 5 次错误验证码时，前端应显示"尝试次数过多，请重新获取验证码"而不是"Invalid code"
4. **记忆系统测试**：建议清空聊天记录后重新测试 MEM-03~06，以验证不同谓词去重、新事实即时召回、超窗口记忆、否定/纠正处理
5. **计费验证**：需通过 API 端点验证文本/语音聊天的积分扣减逻辑
6. **UGC 完整流程**：API 修复后需测试 UGC-07~18（头像上传、预览发布、编辑、删除等）
