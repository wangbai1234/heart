# 09 无障碍规范（Accessibility）— 颜色系统色板

> 该资产为静态规范画布，但其定义的颜色系统将应用于整个 yuoyuo App。  
> 本文件从两个维度分析无障碍：  
> 1. 本色板规范文档自身的无障碍性  
> 2. 色板中各颜色在实际 App 中应用时的无障碍要求

---

## 1. 颜色对比度分析（Color Contrast）

### WCAG 2.1 标准要求

| 等级 | 普通文字（<18px）| 大文字（≥18px 或 ≥14px Bold）| 图形 / 图标 |
|------|----------------|--------------------------|----------|
| AA | 4.5:1 | 3:1 | 3:1 |
| AAA | 7:1 | 4.5:1 | — |

---

### 主色系 Primary 对比度分析

文字颜色 `#3A3A4A`（主文字）与各色阶背景对比：

| 背景色 | HEX | 估算对比度 | WCAG AA（普通文字）| 说明 |
|--------|-----|----------|------------------|------|
| primary-50 | `#FFF1F3` | 约 17:1（估算值） | ✅ 通过 | 浅粉背景，文字高度可读 |
| primary-100 | `#FFE0E6` | 约 14:1（估算值） | ✅ 通过 | |
| primary-200 | `#FFC7D2` | 约 10:1（估算值） | ✅ 通过 | |
| primary-300 | `#FFADC0` | 约 7.5:1（估算值） | ✅ 通过 | |
| primary-400 | `#FFB7C5` | 约 6.5:1（估算值） | ✅ 通过 | 品牌主色背景可用于卡片 |
| primary-500 | `#FF95AA` | 约 4.8:1（估算值） | ✅ 通过（边界值） | |
| primary-600 | `#FF7691` | 约 3.5:1（估算值） | ⚠️ 普通文字不通过 | 仅可用于大文字或图标 |
| primary-700 | `#FF5B7D` | 约 2.8:1（估算值） | ❌ 不通过 | 不可直接在此色背景上放文字 |
| primary-800 | `#FF4369` | 约 2.4:1（估算值） | ❌ 不通过 | 同上 |
| primary-900 | `#E7335A` | 约 3.5:1（估算值） | ⚠️ 仅大文字通过 | 建议配白色文字 |

**推荐**：primary-600 及以下色阶作为背景时，改用**白色文字 `#FFFFFF`**（对比度约 4.5+）

### 辅色系 Secondary 对比度分析（`#3A3A4A` 文字）

| 背景色 | 估算对比度 | WCAG AA |
|--------|----------|---------|
| secondary-50 到 400 | 约 6:1–17:1（估算值） | ✅ 通过 |
| secondary-500 到 600 | 约 4.5:1（估算值） | ✅ 边界 |
| secondary-700 到 900 | 约 2:1–3.5:1（估算值） | ❌/⚠️ 建议白色文字 |

---

### 语义色对比度分析

| 卡片背景 | 文字颜色 | 估算对比度 | 结论 |
|---------|---------|----------|------|
| `#B6E2C7`（成功绿） | `#3A3A4A` | 约 7:1（估算值） | ✅ 通过 |
| `#FFD3A5`（警告橙） | `#3A3A4A` | 约 6:1（估算值） | ✅ 通过 |
| `#F4A6A6`（错误红） | `#3A3A4A` | 约 5.5:1（估算值） | ✅ 通过 |
| `#B6C7F4`（信息蓝） | `#3A3A4A` | 约 6.5:1（估算值） | ✅ 通过 |

所有语义色卡片在炭灰文字下均通过 WCAG AA。

---

### 玻璃叠加层对比度风险

| 叠加透明度 | 底层若为渐变 | 文字可读性 |
|----------|-----------|----------|
| 35% | 底层强烈透出，背景不稳定 | ⚠️ 谨慎：对比度依赖底层色，不稳定 |
| 55% | 底层部分透出 | ⚠️ 需逐案验证，推荐配 `#3A3A4A` 文字 |
| 75% | 接近白色 | ✅ 通常安全 |
| 90% | 接近纯白 | ✅ 安全，与标准白背景等同 |

**规范**：玻璃叠加组件必须保证文字内容区域的对比度 ≥ 4.5:1（AA），建议使用 75% 或 90% 档作为文字容器。

## 2. 点击区域规范（Touch Target）

适用于将色板颜色应用于按钮、图标等可交互元素时：

| 元素类型 | 最小尺寸要求 | iOS HIG 推荐 | WCAG 2.5.8 推荐 |
|---------|-----------|-------------|---------------|
| 普通按钮 | 44 × 44 pt | 44 × 44 pt | 24 × 24 px（物理可触摸区）|
| 图标按钮 | 44 × 44 pt | 44 × 44 pt | 同上 |
| 语义图标（色板中） | 约 30 px（估算值） | ⚠️ 不足，实际使用时需扩大点击区 | — |
| 色阶色块 | 约 128 × 36 px（估算值） | ⚠️ 高度不足，触摸区建议 44 px | — |

**规范**：色板中的色阶色块若作为交互元素（如颜色选择器），必须将可点击区域扩展至最小 44 × 44 pt。

## 3. 焦点状态（Focus State）

适用于 Web / Desktop 端色板工具或颜色选择器：

- 焦点环颜色：使用 `#A7C7E7`（secondary-400）或 `#C8B6FF`（accent-400）
- 焦点环宽度：2–3 px（推荐 3 px）
- 焦点环偏移：2 px（`outline-offset: 2px`）
- 焦点环形状：跟随元素圆角（`border-radius` 一致）
- 禁止：`outline: none` 不得出现在可交互元素上（除非自定义了等价的焦点指示符）

## 4. 文字可读性（Text Readability）

### 本色板文档中的文字可读性

| 文字内容 | 字号 | 背景 | 颜色 | 可读性 |
|---------|------|------|------|-------|
| 页面主标题 | ~38 px | `#FFF8F3` | `#3A3A4A` | ✅ 优秀 |
| 色系列标题 | ~16 px | `#FFF8F3` | `#3A3A4A` | ✅ 优秀 |
| 色阶标签 | ~15 px | `#FFF8F3` | `#7A7873` | ✅ 通过（估算对比度约 4.5:1） |
| HEX 值 | ~13 px | `#FFF8F3` | `#3A3A4A` | ✅ 通过 |
| 玻璃卡片说明 | ~14 px | 渐变背景 | `#7A7873` | ⚠️ 背景不稳定，建议配更深色 |
| 语义色名称 | ~15 px | 语义色卡片背景 | `#3A3A4A` | ✅ 通过（见上方对比度分析） |

### 最小字号规范

- 移动端最小显示字号：**12 px**（低于此字号用户体验差）
- 本色板中最小字号约 13 px，合规
- HEX 值文字建议使用等宽字体（Monospace），避免字符宽度不一致导致视觉抖动

## 5. ARIA 建议（ARIA Recommendations）

适用于将色板颜色应用于实际 App 组件时：

### 颜色信息传达
- **规范**：颜色信息不可作为唯一的传达方式（即色觉异常用户无法仅凭颜色区分状态）
- **实现**：语义色（成功/警告/错误/信息）必须同时配备图标和文字，不能仅靠颜色区分

### 语义色卡片
```
aria-label="成功状态：B6E2C7"
role="img"（仅展示时）
role="button"（可交互时）
```

### 色阶色块
```
aria-label="primary-400 樱花粉 #FFB7C5"
title="#FFB7C5"（Tooltip 内容）
```

### 玻璃叠加卡片
```
aria-label="35% 白色叠加玻璃效果 rgba(255,255,255,0.35)"
role="img"
```

### 徽标（Base Badge）
```
aria-label="品牌基准色 400"
```

## 6. 键盘导航（Keyboard Navigation）

适用于 Web 版色板工具：

| 按键 | 行为 |
|------|------|
| `Tab` | 在色阶行间移动焦点（顺序：从左色柱到右色柱，从上到下） |
| `Shift+Tab` | 反向移动 |
| `Enter` 或 `Space` | 复制当前焦点色块的 HEX 值到剪贴板 |
| `Arrow Up/Down` | 在同一色柱内上下移动色阶 |
| `Arrow Left/Right` | 在色柱间左右切换 |
| `Escape` | 关闭 Tooltip / 取消选中 |

## 7. 屏幕阅读器（Screen Reader）

| 元素 | 推荐朗读内容 | 实现方式 |
|------|-----------|---------|
| 色阶色块 | "主色 400，樱花粉，HEX FFB7C5" | `aria-label` |
| 语义色卡片 | "成功状态颜色，浅绿色，HEX B6E2C7" | `aria-label` |
| 玻璃叠加卡片 | "35% 透明度玻璃叠加效果" | `aria-label` |
| 页面标题 | "yuoyuo 颜色系统" | `<h1>` 标签 |
| 色系区块 | "主色系，樱花粉" | `<section aria-label>` 或 `<h2>` |

**注意**：颜色名称应朗读中文名（"樱花粉"）而非英文（"cherry blossom pink"），以符合中文用户习惯。

## 8. 移动端操作建议

| 建议 | 说明 |
|------|------|
| 触摸区扩大 | 色阶色块触摸区至少 44 pt 高度 |
| 长按呼出详情 | 长按色块显示完整色值（HEX / RGB / HSL），优化单手操作 |
| 避免密集触控 | 色阶行间距不低于 8 pt，防止误触相邻行 |
| 拖拽排序 | 若支持颜色排序功能，拖拽手柄应有足够大的触摸区域 |
| 反馈震动 | 复制 HEX 值成功时，提供轻微 Haptic Feedback（iOS .light） |

## 9. 色觉异常（Color Blindness）支持

| 问题 | 受影响色对 | 解决方案 |
|------|----------|---------|
| 红绿色盲（Deuteranopia） | 成功绿 `#B6E2C7` vs 错误红 `#F4A6A6` | 必须配图标（✓ vs ✕），不可仅靠颜色区分 |
| 蓝黄色盲（Tritanopia） | 信息蓝 `#B6C7F4` vs 警告橙 `#FFD3A5` | 同上，图标必须配套 |
| 全色盲（Achromatopsia） | 所有彩色信息丢失 | 语义信息必须通过文字传达 |

**测试建议**：使用 Stark / Figma Accessibility 插件 / iOS 系统模拟器中的"模拟色觉异常"功能，验证色板在各类色觉缺陷模式下的可辨识度。

## 10. 动效无障碍

适用于应用色板颜色的动画效果：

- 遵守 `prefers-reduced-motion` 媒体查询：当用户设置"减少动效"时，禁用所有装饰性动画
- 色阶行的 stagger 动画在 `prefers-reduced-motion: reduce` 时改为静态显示
- Skeleton 闪烁动画在减少动效模式下改为静态灰色占位

```css
/* 建议规范（非代码实现，仅作规则说明） */
@media (prefers-reduced-motion: reduce) {
  /* 所有动画 duration 设为 0 或极小值 */
  /* 移除 transform/opacity 过渡 */
}
```
