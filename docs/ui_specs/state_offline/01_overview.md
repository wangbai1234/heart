# 01 Overview — 离线降级 / 弱网状态

## 页面名称
离线降级状态页 (Offline / Weak-Network Degraded State)

内部 ID: `state_offline` | Phase 3 — State | 设计稿文件: `17_state_offline.png`

---

## 作用
在 WebSocket 连接 3 次重连全部失败后，将聊天界面无损降级为"离线模式"——用户仍可继续输入消息，消息被本地暂存，连接恢复后自动补发。整个过程零中断感，保持对话温度。

---

## 目标
1. 告知用户当前网络异常，但不触发用户恐慌
2. 让用户明白消息已暂存、连接恢复后会自动发送
3. 保留 Composer 输入能力，维持对话连续性
4. 提供手动"重试"逃生口

---

## 触发条件
| 事件 | 具体值 |
|------|--------|
| WebSocket 重连失败次数 | ≥ 3 次 |
| 触发延迟 | 第 3 次失败后立即 |
| 适用场景 | 地铁隧道、飞行模式、弱 Wi-Fi 断联 |

---

## 退出方式
| 方式 | 操作 |
|------|------|
| 自动恢复 | WebSocket 重新连接成功 → 横幅消失、消息自动发送、暂存分割线消失 |
| 手动重试 | 点击横幅右侧「重试」按钮 → 立即触发一次重连 |
| 离开页面 | 返回上一级（不影响暂存消息） |

---

## 用户流程
```
正常聊天
  ↓ 信号断开
  ↓ 后台静默重连 × 3
  ↓ 均失败
进入离线状态
  ├─ Nav Bar 副标题变为「离线 · 消息将自动发送」
  ├─ 软横幅出现（云朵图标 + 文字 + 重试按钮）
  ├─ 历史消息区域整体透明度降至 ~92% 暗淡
  ├─ Composer 时钟角标出现
  └─ "以下为暂存消息" 分割线出现（尚无暂存消息时可不显示内容）
用户继续输入
  └─ 消息以"已暂存"状态显示在暂存区（分割线以下）
用户点击「重试」或网络自动恢复
  └─ 退出离线状态，暂存消息自动发出
```

---

## 设计理念
- **零打断原则**：横幅是"软提示"而非模态弹窗，用户可继续输入
- **温暖托底**：副标题"消息将自动发送"是情绪安抚，减少焦虑
- **层次清晰**：横幅 > 历史消息区（暗淡）> 暂存分割线 > Composer，信息层级一目了然
- **二次元治愈**：粉色云朵图标、柔软措辞（"网络不太稳定"而非"连接失败"）保持品牌温度

---

## 视觉关键词
`软横幅` `云朵图标` `历史消息暗淡` `分割线暂存提示` `时钟角标` `暖粉色` `glassmorphism` `治愈系` `低焦虑`

---

## 情绪传达（重点）
**用户在此状态下的目标情绪：安心 + 不焦虑 + 信任**

| 设计手段 | 情绪效果 |
|----------|----------|
| 副标题「消息将自动发送」 | 承诺 → 消除不确定感 |
| 云朵 emoji 图标（粉色，非警告红） | 温柔警示 → 不触发恐慌 |
| "网络不太稳定，刚才的消息已暂存" | 事实陈述 + 行动结果确认 → 安心 |
| 「重试」按钮（可选，非强迫） | 控制感 → 用户感到有主动权 |
| 历史消息依然可见（仅微暗淡） | 连续性保留 → 无失联割裂感 |
| Composer 仍可输入 | 能动性保留 → 不被"锁死" |
| 时钟角标（非禁止符号） | 等待感 → 非错误感 |
| 分割线文字「网络恢复后会自动发送」 | 再次承诺 → 二次安心 |
