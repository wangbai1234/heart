# 09 无障碍规范 — App Icon（多色版 + 单色版）

## 说明

App Icon 作为系统级视觉资源，其无障碍需求与界面组件不同。本文件涵盖图标本身的无障碍设计规范，以及图标在 App 内部使用场景的无障碍要求。

---

## 点击区域（Touch Target）

### 桌面图标

| 规格 | 值 |
|------|-----|
| iOS 主屏幕图标尺寸 | 60 × 60 pt（@2x: 120px，@3x: 180px） |
| iOS 系统实际点击热区 | 约 76 × 76 pt（系统自动扩展，估算值） |
| Android 桌面图标尺寸 | 48 × 48 dp（mdpi 基准） |
| 最小建议点击区域 | 44 × 44 pt（符合 Apple HIG 标准） |
| 评估 | 图标尺寸满足最小点击区域要求，系统自动管理热区扩展 |

### App 内部图标使用

当图标作为按钮、导航元素在 App 内出现时：

| 规格 | 建议值 |
|------|--------|
| 最小点击热区 | 44 × 44 pt（Apple HIG）/ 48 × 48 dp（Material） |
| 若图标视觉尺寸小于热区 | 使用 padding 或透明热区扩展 |
| 热区与视觉边界不一致时 | 不影响视觉，仅扩展可交互区域 |

## 颜色对比度

### 全色渐变版

| 对比场景 | 前景色 | 背景色 | 对比值（估算值）| 评估 |
|---------|--------|--------|--------------|------|
| 图标 vs 展示画布背景 | 混合粉蓝紫 | `#FAE8D8` | 无需对比（图标为图形，非文字）| — |
| 图标内星光 vs 主体 | `rgba(255,255,255,0.85)` | `#C8B6FF`（紫色区域）| ≈ 1.8:1（估算值）| 装饰性元素，无障碍标准豁免 |
| 图标内云朵 vs 主体 | `rgba(255,255,255,0.60)` | `#A7C7E7`（蓝色区域）| ≈ 2.1:1（估算值）| 装饰性元素，豁免 |

**结论**：图标为纯图形资源，不包含文字，不受 WCAG AA 文字对比度（4.5:1）限制。所有内部元素均为装饰性，符合 WCAG 1.4.11（非文字对比度）豁免条件。

### 单色版

| 对比场景 | 评估 |
|---------|------|
| 图标 vs 暗色背景 | 单色版采用暖灰色系，与纯黑背景对比度约 4.2:1（估算值），略低于 AA 标准，但作为图形资源豁免 |
| iOS Tinted Mode | 系统会根据用户色彩设置自动调整，实际对比度由系统保证 |

## 焦点状态（Focus State）

### 桌面图标

| 场景 | 处理方式 |
|------|---------|
| iOS 键盘辅助功能（Switch Control） | 系统自动在图标周围绘制焦点环（蓝色矩形，带圆角） |
| iOS VoiceOver | 系统自动朗读图标名称，无需额外实现 |
| Android TalkBack | 系统自动绘制焦点矩形 |

**研发工作量**：无需实现，系统自动处理。

### App 内部图标使用焦点状态

如图标用于可交互组件（按钮等）：
- iOS: 使用系统焦点环，或自定义 `UIFocusRingType`
- Web: 提供 `:focus-visible` 样式，建议使用 `2px solid #C8B6FF`（薰衣草紫，与品牌色匹配）
- 焦点环不应遮盖图标关键区域

## 文本可读性

N/A — 图标内无文字元素。

### App 名称文字（外部场景）

当「yuoyuo」文字与图标组合出现（如 App Store 截图、Splash Screen）时：

| 规格 | 建议值 |
|------|--------|
| 文字颜色 | `#3A3A4A`（品牌 Ink 色）对白底，对比度 ≈ 11:1（满足 AAA） |
| 最小字号 | 14pt（移动端）/ 16px（Web）|
| 字重 | Medium 500 或 SemiBold 600 |
| 背景 | `#FFF8F3`（品牌 Surface 色）或白色 |

## ARIA 建议

### 图标用于 `<img>` 标签时

| 场景 | 建议属性 |
|------|---------|
| 纯装饰性图标 | `alt=""` 或 `role="presentation"` |
| 作为应用标志（Logo） | `alt="yuoyuo 应用图标"` |
| 作为导航链接图标 | `alt=""` + 父级 `<a>` 提供 `aria-label="返回首页"` |

### 图标用于 `<button>` 时

| 建议 | 说明 |
|------|------|
| `aria-label` | 在按钮上提供语义标签，如 `aria-label="打开 yuoyuo"` |
| `title` | 提供 tooltip 文字（桌面 Web 场景） |
| 不依赖颜色区分状态 | 若有 active/inactive 状态，除颜色外还需提供形态变化或文字提示 |

---

## Keyboard Navigation（键盘导航）

### 桌面图标

| 场景 | 说明 |
|------|------|
| iOS Switch Control | 系统自动支持，图标可通过外接键盘 Tab 键聚焦 |
| macOS Catalyst | 如果 App 移植到 macOS，图标按钮应支持 Return/Space 键激活 |
| Web App | 图标作为链接时应确保 Tab 可聚焦，Enter 可激活 |

## Screen Reader（屏幕朗读）支持

### iOS VoiceOver

| 属性 | 规范 |
|------|------|
| 图标朗读内容 | 桌面：「yuoyuo」（App 名称，由 Info.plist `CFBundleDisplayName` 控制）|
| 图标类型提示 | VoiceOver 自动朗读「应用程序」类型 |
| 研发工作量 | 设置正确的 `CFBundleDisplayName` 为「yuoyuo」即可 |

### Android TalkBack

| 属性 | 规范 |
|------|------|
| 图标朗读内容 | `android:label` 值（建议设为「yuoyuo」）|
| 图标描述 | 可通过 `android:accessibilityDescription` 补充（非必须） |

### Web

| 场景 | 建议 |
|------|------|
| 图标 `<img>` | `alt="yuoyuo"` |
| 图标 SVG | `<title>yuoyuo 应用图标</title>` + `aria-labelledby` |

## 移动端操作建议

### 手势操作

| 操作 | 系统行为 | 无障碍备注 |
|------|---------|-----------|
| 单次点击 | 打开 App | 无需额外处理 |
| 长按 | 触发上下文菜单（Haptic Touch） | 系统自动处理振动反馈 |
| 双指点击（VoiceOver 激活时） | 等同于单击 | 系统自动处理 |
| 三指轻扫（VoiceOver 导航） | 在图标间移动 | 系统自动处理 |

### 视觉辅助功能适配

| 系统功能 | 图标适配建议 |
|---------|------------|
| 增大文字（Larger Text） | 不影响图标（图标尺寸固定）|
| 高对比度模式（Increase Contrast） | 单色版可作为高对比度备选 |
| 减少透明度（Reduce Transparency） | 图标为不透明 PNG，不受影响 |
| 减少动态效果（Reduce Motion） | 任何图标动效须响应此设置，关闭时退化为静态 |
| 深色模式（Dark Mode） | 建议提供深色版图标（见 A-10 规范），当前设计以浅色为主 |
| iOS Tinted Icon（iOS 16+） | 单色版专为此场景设计，颜色由系统接管 |

## 无障碍合规评估

| WCAG 2.1 标准 | 适用性 | 评估结果 |
|-------------|--------|---------|
| 1.1.1 非文字内容 | 适用（图标用于界面时）| 需提供 `alt` 文字 |
| 1.4.1 不依赖颜色 | 适用 | 图标形状本身（心形+气泡）已提供颜色外的语义，满足 |
| 1.4.3 对比度（文字）| 不适用（无文字）| — |
| 1.4.11 非文字对比度 | 部分适用 | 图标与背景对比度须满足 3:1，建议测试各背景色场景 |
| 2.1.1 键盘可访问 | 适用（App 内交互场景）| 系统自动处理，App 内需确保 |
| 2.4.7 焦点可见 | 适用（App 内）| 系统焦点环满足 |
