# 09 无障碍规范（Accessibility）

## 说明

本文件基于 WCAG 2.1 AA 标准，结合 yuoyuo 移动端 AI 伴侣应用的实际场景，提出动效分镜图所涉及组件的无障碍设计建议。

---

## 点击区域（Touch Target）

### 规范要求

iOS HIG 最小可点击区域：44×44 pt

| 组件 | 视觉尺寸（估算） | 点击区域建议 | 是否达标 |
|------|----------------|-------------|---------|
| 返回按钮 `<` | ~24×24pt | 需扩展至 44×44pt | 需调整 |
| 菜单按钮 `···` | ~24×24pt | 需扩展至 44×44pt | 需调整 |
| 麦克风按钮 🎤 | ~24×24pt | 需扩展至 44×44pt | 需调整 |
| 加号按钮 `+` | ~36×36pt（估算） | 扩展至 44×44pt | 需调整 |
| 消息气泡（可选中） | 内容宽度不固定 | 气泡区域可满足 | 达标 |
| 情绪球（可点击？） | ~36pt | 建议扩展至 44×44pt | 需确认是否可交互 |

**实现建议**：对视觉尺寸小于 44pt 的可交互元素，使用 padding 扩展点击区域，或使用透明叠层（hit-slop）扩大响应范围。

## 颜色对比度（Color Contrast）

### 关键文字对比检查

基于 WCAG 2.1 AA 标准：正常文字 ≥4.5:1，大文字（≥18pt 或 ≥14pt Bold）≥3:1

| 元素 | 前景色 | 背景色 | 对比度（估算） | 是否达标 |
|------|--------|--------|--------------|---------|
| 消息正文（AI气泡） | #3A3A4A | #FFFFFF | ~9.5:1 | ✅ 达标 |
| 消息正文（用户气泡） | #3A3A4A | #FFD6DF | ~6.5:1（估算） | ✅ 达标 |
| 时间戳文字 | #AAAAAA | #FFFFFF | ~2.3:1（估算） | ⚠️ 不达标（装饰性文字，可接受） |
| 导航栏昵称 | #3A3A4A | #FFFFFF | ~9.5:1 | ✅ 达标 |
| "在线"状态文字 | #7DCE7D | #FFFFFF | ~2.1:1（估算） | ⚠️ 不达标 |
| 占位文字 | #BBBBCC | #FFFFFF/浅灰 | ~2.5:1（估算） | ⚠️ 不达标（占位文字豁免，可接受） |
| 动效说明文字 | #6B6B7A | #FFF8F3 | ~5.8:1（估算） | ✅ 达标 |
| 动效标题 | #3A3A4A | #FFF8F3 | ~8.8:1（估算） | ✅ 达标 |

**改进建议**：
- "在线"绿色文字（#7DCE7D）建议加深至 #4CAF50（约 3.5:1）以改善可读性
- 时间戳文字属于辅助信息，颜色不达标可接受，但建议为 VoiceOver 用户提供 accessibilityLabel

## 焦点状态（Focus State）

### 键盘导航（Web 端）

| 组件 | 焦点样式建议 |
|------|-------------|
| 返回按钮 | `outline: 2px solid #FFB7C5; outline-offset: 2px` |
| 菜单按钮 | 同上 |
| 加号按钮 | `outline: 2px solid #FFB7C5; outline-offset: 2px; border-radius: 50%` |
| 麦克风按钮 | 同返回按钮 |
| 输入框 | `box-shadow: 0 0 0 2px #FFB7C5;` |
| 消息气泡（可交互时） | 浅粉色外框 |

**注意**：分镜图未定义焦点样式，以上为规范建议。

## 文本可读性（Text Legibility）

| 规范项 | 状态 | 备注 |
|--------|------|------|
| 最小字号 ≥11px | ✅ | 时间戳约 11px，刚达到最小值 |
| 行高 ≥1.5（正文） | ✅（估算） | 气泡正文行高约 1.6 |
| 正文字号 ≥14px | ✅ | 消息正文约 14px |
| 避免全大写正文 | ✅ | 无全大写设计 |
| 避免过长单行文本 | ✅ | 气泡有最大宽度限制 |

## ARIA 建议

### 主要组件 ARIA 属性

| 组件 | 建议 ARIA 属性 |
|------|--------------|
| 返回按钮 | `role="button"` `aria-label="返回"` |
| 菜单按钮 | `role="button"` `aria-label="更多选项"` |
| 情绪球（仅展示） | `role="img"` `aria-label="悠悠的情绪状态"` |
| 情绪球（可点击） | `role="button"` `aria-label="查看悠悠情绪详情"` |
| AI 消息气泡 | `role="listitem"` `aria-label="悠悠说：{消息内容}"` |
| 用户消息气泡 | `role="listitem"` `aria-label="你说：{消息内容}"` |
| 时间戳 | `aria-label="{时间}"` 或 `aria-hidden="true"`（辅助信息） |
| 已读回执 ✓ | `aria-label="已送达"` |
| 麦克风按钮 | `role="button"` `aria-label="语音输入"` |
| 加号按钮 | `role="button"` `aria-label="更多功能"` |
| 输入框 | `role="textbox"` `aria-label="消息输入"` `aria-placeholder="输入消息..."` |
| 消息列表 | `role="log"` `aria-live="polite"` `aria-label="聊天记录"` |
| 语音波 | `role="img"` `aria-label="悠悠正在说话"` |

## 屏幕阅读器（Screen Reader）支持

### iOS VoiceOver

| 场景 | 建议 |
|------|------|
| 新消息到达 | 使用 `aria-live="polite"` 通知屏幕阅读器，不打断用户当前操作 |
| 情绪球脉冲动效 | 添加 `aria-hidden="true"` 跳过视觉动效，提供静态文字描述 |
| 语音波动效 | `aria-label="AI 正在语音播放"` 或使用 `role="status"` |
| 页面过渡 | 过渡完成后聚焦到新页面第一个有意义元素 |

### Android TalkBack

- 建议参照 iOS VoiceOver 规范，确保 `contentDescription` 属性覆盖所有可交互元素

## 键盘导航（Keyboard Navigation — Web 端）

| 导航顺序 | 元素 | Tab 顺序 |
|---------|------|---------|
| 1 | 返回按钮 | Tab 1 |
| 2 | 情绪球（如可交互） | Tab 2 |
| 3 | 菜单按钮 | Tab 3 |
| 4 | 消息列表中的消息（可选中时） | Tab 4+ |
| 最后 | 输入框 | Tab N-1 |
| 最后 | 加号按钮 | Tab N |

- `Enter` / `Space`：激活聚焦按钮
- `Escape`：关闭弹出层（菜单等）
- 输入框内 `Enter`：发送消息（需明确定义，**设计稿未定义**）

## 移动端操作建议

### 手势建议

| 手势 | 功能 | 说明 |
|------|------|------|
| 单指点击 | 激活按钮、选择消息 | 标准操作 |
| 长按 | 消息操作菜单（复制/删除等） | **设计稿未定义** |
| 下滑（在聊天页 Sheet 中） | 关闭聊天页返回首页 | 对应页面过渡反向 |
| 双指放大 | 放大媒体内容 | **设计稿未定义** |

### 单手操作优化

- 关键操作按钮（加号、麦克风）位于拇指可达区域（屏幕底部）✅
- 导航栏返回按钮位于左上角，单手操作需伸展（建议支持右滑返回手势）

### 动效无障碍

| 规范 | 建议 |
|------|------|
| 减少动态效果（Reduce Motion） | 检测 `prefers-reduced-motion`，禁用 scale 动效，保留 opacity |
| 闪烁频率 | 情绪球脉冲频率约 0.83Hz（1200ms），远低于光敏触发阈值（≥3Hz），安全 |
| 自动播放动效 | 情绪球和语音波为自动播放，建议提供关闭选项 |
