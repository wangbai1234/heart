# 09 Accessibility — 登录页 Login

---

## 触摸目标尺寸（Touch Target Size）

iOS HIG 标准：最小触摸目标 ≥ 44×44pt（逻辑像素）

| 元素 | 视觉尺寸（估算值） | 触摸区域 | 是否达标 |
|------|-----------------|---------|---------|
| 「发送登录链接」主按钮 | 宽 ~305px × 高 ~43px | 自身即为大目标 | ⚠️ 高度估算值接近 44px，建议确保 min-height: 44pt（实现时应 ≥44px，目视约达标） |
| 《用户协议》链接 | 约 72px × 16px（估算值） | 原始尺寸不足 | ❌ 需通过 padding（上下各 12px）或 `hitSlop` 扩展至 ≥ 44×44pt |
| 《隐私政策》链接 | 约 72px × 16px（估算值） | 原始尺寸不足 | ❌ 同上，需扩展触摸区域 |
| 「我有兑换码，直接激活 →」 | 约 ~220px × 22px（估算值） | 原始高度不足 | ❌ 需通过 padding 上下各 11px 或整行可点击区域 ≥44pt 高 |
| 邮箱输入框 | 宽 ~305px × 高 ~37px（估算值） | 需确保 ≥44pt 高 | ⚠️ 估算值偏低，建议 min-height: 44pt |

**建议**：所有可交互元素使用 `paddingVertical` / `hitSlop` / `minHeight: 44` 确保达标，不依赖视觉尺寸。

## 颜色对比度（Color Contrast — WCAG AA/AAA）

WCAG AA 标准：正文 ≥ 4.5:1，大文字（≥18pt 或 14pt Bold）≥ 3:1
WCAG AAA 标准：正文 ≥ 7:1

| 元素 | 前景色 | 背景色 | 估算对比度 | WCAG AA | WCAG AAA |
|------|--------|--------|----------|---------|---------|
| "yuoyuo" 字标 | `#3A3A4A` | `#FFF8F3` | ~11.5:1（估算值） | ✅ 通过 | ✅ 通过 |
| Tagline 文字 | `#8A8898` | `#FFF8F3` | ~4.1:1（估算值） | ⚠️ 接近临界，建议验证实际值 | ❌ |
| 说明文字（HintText） | `#8A8898` | `rgba(255,255,255,0.75)` | ~3.8:1（估算值） | ⚠️ 小字体可能不足，建议加深至 `#6B6880` | ❌ |
| 主按钮文字 | `#FFFFFF` | `#FFB7C5`（渐变起点） | ~2.1:1（估算值） | ❌ 白字在粉色底上对比度不足（已知设计挑战）——建议文字改用 `#5C2030` 深玫红，或按钮色加深 |
| 协议基础文字 | `#8A8898` | `#FFF8F3` | ~4.1:1（估算值） | ⚠️ 临界 | ❌ |
| 协议链接文字 | `#FFB7C5` | `#FFF8F3` | ~1.9:1（估算值） | ❌ 严重不足——链接仅靠颜色区分时需同时加下划线 | ❌ |
| 兑换码文字 | `#3A3A4A` | `#FFF8F3` | ~11.5:1（估算值） | ✅ 通过 | ✅ 通过 |
| 占位文字 | `#B0A8B4` | `rgba(255,255,255,0.75)` | ~2.4:1（估算值） | ❌ 占位文字不要求 AA，但建议不低于 3:1 |

**关键问题**：
1. 主按钮白字 vs 粉色背景对比度偏低（约 2.1:1），建议按钮文字使用深色或加强按钮背景色深度
2. 协议链接颜色（粉色 on 奶油白）对比度严重不足，必须添加下划线作为非颜色区分符

## 焦点状态（Focus State）

| 元素 | 建议焦点样式 |
|------|------------|
| 邮箱输入框 | 底部分隔线变为 2px solid `#FFB7C5`（主粉色）；或整体输入行加粉色边框（1px） |
| 主按钮 | `outline: 3px solid #C8B6FF`（薰衣草紫焦点环，offset: 2px） |
| 《用户协议》链接 | `outline: 2px solid #FFB7C5`，underline 加粗 |
| 《隐私政策》链接 | 同上 |
| 兑换码链接 | `outline: 2px solid #3A3A4A`，offset: 2px |

### Tab 顺序建议
```
1. 邮箱输入框（EmailInput）
2. 「发送登录链接」按钮（PrimaryButton）
3. 《用户协议》链接（AgreementLink）
4. 《隐私政策》链接（PrivacyLink）
5. 「我有兑换码，直接激活」链接（RedeemLink）
```

## ARIA 建议（适用于 Web 实现，React Native 使用 accessibilityRole/accessibilityLabel）

| 元素 | ARIA Role | aria-label 建议 |
|------|-----------|---------------|
| 页面根容器 | `main` | — |
| "yuoyuo" 字标 | `img`（若为图片）/ `heading`（若为文字，level 1） | `"yuoyuo - AI 伴侣应用"` |
| Tagline | `paragraph` | — |
| 邮箱输入框 | `textbox` | `aria-label="邮箱地址"` `aria-required="true"` `type="email"` |
| 说明文字 | `note` / `paragraph` | — |
| 主按钮 | `button` | `aria-label="发送登录链接到邮箱"` `aria-busy="true"（Loading 时）` |
| 《用户协议》链接 | `link` | `aria-label="查看用户协议"` |
| 《隐私政策》链接 | `link` | `aria-label="查看隐私政策"` |
| 兑换码链接 | `link` / `button` | `aria-label="使用兑换码直接激活"` |
| 玻璃心形图标 | `img` | `aria-label="yuoyuo 爱心图标"` |
| 插画 | `img` | `aria-hidden="true"`（纯装饰性，屏幕阅读器跳过） |

## Keyboard Navigation

| 操作 | 行为 |
|------|------|
| `Tab` | 按 Tab 顺序聚焦可交互元素（输入框→按钮→链接→链接→兑换码） |
| `Shift+Tab` | 反向 Tab 导航 |
| `Enter`（输入框聚焦时） | 触发「发送登录链接」（相当于点击主按钮）|
| `Enter`（按钮聚焦时） | 触发按钮点击 |
| `Enter`（链接聚焦时） | 打开链接目标 |
| `Space`（按钮聚焦时） | 等效于 Enter，触发点击 |
| `Escape` | 收起键盘（移动端）；关闭任何弹出的 Modal/Sheet |

## 移动端操作友好度

### 单手操作分析（右手拇指）
| 区域 | 可达性 | 建议 |
|------|--------|------|
| 状态栏 | ✅ 顶部，非可交互 | — |
| 插画区 | ✅ 纯展示 | — |
| "yuoyuo" 字标 | ✅ 纯展示 | — |
| 邮箱输入框 | ✅ 中部，单手可达 | — |

### 拇指舒适区说明
- iPhone 14（390×844pt）：拇指舒适区约为屏幕下 60%（y > 330pt）
- 主要操作（邮箱输入 + 主按钮）均位于舒适区内 ✅
- 次要操作（协议链接、兑换码）位于底部，单手略需伸展但可接受 ⚠️
