# 01 Overview — 反馈组件三联（Toast / Modal / Sheet）

## 页面名称

**反馈组件三联展示**

Phase: Phase 3 — State

---

## 作用与目标

本资产为横版三联组件展示画布（1536×1024），并排呈现应用中三种核心反馈/覆盖层组件：

| 编号 | 组件类型 | 中文描述 |
|------|----------|----------|
| 1 | Toast（顶部气泡通知） | 提示信息 · 2.5秒自动关闭 |
| 2 | Modal（居中确认弹窗） | 莫代尔 · 破坏性但温暖 |
| 3 | Bottom Sheet（底部选项表单） | 底部页面 · 用于选项选择 |

三者均叠加在聊天主页面（悠悠 AI 伴侣对话界面）的背景之上，展示各自的覆盖层语境。

---

## 触发条件

### Toast（组件 1）
- 用户完成某项操作（如：兑换码兑换成功、会员激活）系统自动弹出
- 无需用户主动触发，由业务逻辑事件驱动
- 显示位置：导航栏下方 ≈ 16px 处，水平居中

### Modal（组件 2）
- 用户点击「退出登录」等高风险/破坏性操作按钮时弹出
- 需要用户明确确认或取消方可继续
- 背景叠加半透明深色蒙层（scrim），阻断底层交互

### Bottom Sheet（组件 3）
- 用户点击主题选择、选项配置等需要从列表中选择的功能入口时弹出
- 从屏幕底部向上滑入
- 背景呈现轻度 scrim 遮罩（相比 Modal 更浅）

---

## 退出方式

| 组件 | 退出方式 |
|------|----------|
| Toast | 2.5 秒倒计时后自动消失；或用户上滑/轻触可提前关闭（设计稿未明确标注，视产品决策） |
| Modal | 点击「取消」按钮；点击蒙层外区域关闭（通常为低风险操作不允许，破坏性操作 Modal 仅可「取消」） |
| Bottom Sheet | 点击「完成」按钮提交选择；向下滑动手势关闭；点击蒙层外区域关闭 |

---

## 用户流程

```
用户操作触发 → 组件出现动画（进入）→ 用户感知/交互
→ [Toast] 自动倒计时 2.5s → 淡出消失
→ [Modal] 用户点击「确认退出」→ 执行破坏性操作 → Modal 消失
         用户点击「取消」   → 取消操作 → Modal 消失，返回原页面
→ [Sheet] 用户选择选项（单选） → 点击「完成」→ Sheet 收起，应用设置
```

## 设计理念

### 核心理念：「温柔约束」
- 三种组件均遵循 yuoyuo 的「二次元疗愈」视觉语言，在功能性约束（覆盖、打断用户流）与情感温度之间取得平衡
- 即使是破坏性操作（退出登录）的 Modal，也用粉色圆角卡片 + 云朵插图软化了警示感，避免 iOS 系统级的冷硬对话框
- Toast 采用白色圆角气泡 + 粉色勾选图标，传递「成功」情绪而非系统通知感

### 层次哲学：Z轴叠加
- 底层：聊天背景（渐变 + 水晶心脏插画）
- 中层：scrim 遮罩（Modal 深色 / Sheet 浅色）
- 顶层：功能组件（Toast / Modal Card / Sheet Card）

## 情绪传达

| 组件 | 情绪引导 |
|------|----------|
| Toast | 「做到了！」— 轻快的成功感，短暂出现不打扰，像好朋友的一个微笑点头 |
| Modal | 「我在确认你真的要这样做」— 温柔但郑重的二次确认，不恐吓，但让用户感受到操作重量 |
| Bottom Sheet | 「来，选一个你喜欢的」— 轻松愉悦的选择过程，底部滑入如同伸出邀请的手 |
