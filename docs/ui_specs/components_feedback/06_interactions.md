# 06 Interactions — 完整交互规范

## 概述

本文档覆盖三个组件（Toast / Modal / BottomSheet）的完整交互行为。
设计稿为静态截图，以下标注中：
- 设计稿明确可见的行为标记为 **[设计稿可见]**
- 行业惯例/产品逻辑推断的行为标记为 **[推断]**
- 待产品明确决策的行为标记为 **[待定]**

---

## 1. Toast 交互

### 触发条件
**[设计稿可见]** 操作完成后由系统自动触发，场景为：
- 兑换码兑换成功 → "兑换成功，会员已激活。"
- 其他操作成功/失败/信息通知（通过 Toast 变体实现）

### 进入动画
**[推断]**
| 属性 | 值 |
|------|-----|
| 动画类型 | 从顶部滑入 + 渐显（slide-down + fade-in） |
| 初始位置 | translateY(-100%) |
| 目标位置 | translateY(0) |
| 初始透明度 | opacity: 0 |
| 目标透明度 | opacity: 1 |
| 时长 | 300ms |
| 缓动曲线 | `cubic-bezier(0.34, 1.56, 0.64, 1)`（弹性进入） |

### 停留阶段
**[设计稿可见]** 停留 2.5 秒（底部标注："2.5秒自动关闭"）

### 退出动画
**[推断]**
| 属性 | 值 |
|------|-----|
| 动画类型 | 向上滑出 + 渐隐（slide-up + fade-out） |
| 目标位置 | translateY(-100%) |
| 目标透明度 | opacity: 0 |
| 时长 | 200ms |
| 缓动曲线 | `cubic-bezier(0.25, 0.46, 0.45, 0.94)` |

### 用户手势
**[推断]** 建议实现：
- 上滑手势（swipe-up）提前关闭 Toast
- 轻触 Toast 提前关闭（可选）

**[待定]** 是否支持点击 Toast 跳转到相关页面（如会员中心）？

### 排队规则
**[推断]** 如多个 Toast 连续触发：
- 方案A：后者等前者消失后再显示（队列模式）
- 方案B：后者替换前者（替换模式）
- 建议：替换模式，避免用户等待

## 2. Modal 交互

### 触发条件
**[设计稿可见]** 用户点击"退出登录"等破坏性操作按钮

### ScrimOverlay 进入
**[推断]**
| 属性 | 值 |
|------|-----|
| 动画类型 | 渐显（fade-in） |
| opacity 变化 | 0 → 0.45 |
| 时长 | 200ms |
| 缓动曲线 | linear |
| 并行关系 | 与 ModalCard 动画同步开始 |

### ModalCard 进入动画
**[推断]**
| 属性 | 值 |
|------|-----|
| 动画类型 | 从中心弹出（scale + fade-in） |
| 初始缩放 | scale(0.85) |
| 目标缩放 | scale(1.0) |
| 初始透明度 | opacity: 0 |
| 目标透明度 | opacity: 1 |
| 时长 | 250ms |
| 缓动曲线 | `cubic-bezier(0.34, 1.56, 0.64, 1)`（轻弹性） |

### 按钮交互 — 确认退出

| 事件 | 行为 |
|------|------|
| Pressed 态 | 按钮亮度降低 10%，scale(0.98)，100ms |
| Released | 执行退出操作，Modal 关闭 |
| Loading（等待后端） | 按钮显示旋转 Spinner，禁止重复点击 |

**[推断]** 执行破坏性操作时建议按钮立即进入 loading 态，防止重复提交。

### 按钮交互 — 取消

| 事件 | 行为 |
|------|------|
| Pressed 态 | 背景色变为 `#F5F5F5`，100ms |
| Released | 关闭 Modal，返回原页面，不执行任何操作 |

### ScrimOverlay 点击
**[待定]** 对于破坏性操作 Modal（退出登录），点击 scrim 是否关闭 Modal？
- 推荐：**允许关闭**（等同于取消），提升可用性
- 不推荐：禁止关闭（强迫用户看完整信息），对于真正高风险操作（删除账号）可考虑

### ModalCard / ScrimOverlay 退出动画
**[推断]**
| 属性 | 值 |
|------|-----|
| ModalCard | scale(0.85) + opacity: 0，150ms |
| ScrimOverlay | opacity: 0，200ms |
| 缓动 | `cubic-bezier(0.25, 0.46, 0.45, 0.94)` |

### 键盘行为
**[推断]** 如 Modal 内有输入框（本设计稿无）：
- 键盘弹出时 Modal 向上平移
- 当前设计稿无输入场景，不适用

## 3. BottomSheet 交互

### 触发条件
**[设计稿可见]** 用户点击主题设置等选项配置入口，底部 Sheet 从下方滑入

### ScrimOverlay 进入
**[推断]**
| 属性 | 值 |
|------|-----|
| 动画类型 | 渐显 |
| opacity 变化 | 0 → 0.25 |
| 时长 | 200ms，与 Sheet 同步 |

### BottomSheet 进入动画
**[推断]**
| 属性 | 值 |
|------|-----|
| 动画类型 | 从底部向上滑入（slide-up） |
| 初始位置 | translateY(100%) |
| 目标位置 | translateY(0) |
| 时长 | 350ms |
| 缓动曲线 | `cubic-bezier(0.32, 0.72, 0, 1)`（iOS-style spring） |

### RadioOption 交互

| 事件 | 行为 |
|------|------|
| 点击选项 | 当前选中项 Radio 状态切换为选中（粉色填充），其他项取消选中 |
| 选中动画 | Radio 内圆从 scale(0) 弹出至 scale(1)，150ms，弹性曲线 |
| Pressed 态 | 整行轻微高亮背景 `rgba(255,183,197,0.12)` |

**[设计稿可见]** 当前"浅色"为选中状态。

### "完成" 按钮交互

| 事件 | 行为 |
|------|------|
| Pressed 态 | scale(0.98) + 亮度降 10%，100ms |
| Released | 应用选中的主题设置，Sheet 收起 |
| **[待定]** | 是否显示选中项的即时预览（切换主题动画）？ |

### 拖动手势（Drag Handle）

**[推断]** 建议实现：
| 手势 | 行为 |
|------|------|
| 向下拖动少量（< 30%高度） | 释放后 Sheet 弹回原位（snap back） |
| 向下拖动超过 30% 高度 | 释放后 Sheet 收起关闭 |
| 快速向下 flick | 立即关闭 Sheet |
| 向上拖动 | 可扩展至全屏（若内容支持）或 snap 回原高度 |

### ScrimOverlay 点击
**[推断]** 点击 scrim 关闭 Sheet（取消选择，不应用任何更改）

### BottomSheet / ScrimOverlay 退出动画
**[推断]**
| 属性 | 值 |
|------|-----|
| Sheet | translateY(100%)，300ms |
| ScrimOverlay | opacity: 0，250ms |
| 缓动 | `cubic-bezier(0.25, 0.46, 0.45, 0.94)` |

### 键盘适应
**[推断]** 当前 Sheet 无输入框，键盘行为不适用。
若未来 Sheet 内增加输入框，Sheet 需配合键盘高度上移。

## 4. 通用手势优先级

| 优先级 | 手势 | 处理方 |
|--------|------|--------|
| 最高 | Toast 上滑关闭 | Toast 自身 |
| 高 | Sheet 拖动关闭 | BottomSheet |
| 中 | Scrim 点击关闭 | Modal / Sheet |
| 低 | 按钮点击 | Modal / Sheet |

## 5. 无障碍交互

### Toast
**[推断]**
- VoiceOver 出现时播报 Toast 内容："兑换成功，会员已激活。"
- 建议使用 `aria-live="assertive"` 确保即时播报

### Modal
**[推断]**
- VoiceOver 焦点应自动移入 Modal，聚焦至标题
- 播报顺序：标题 → 说明 → 确认按钮 → 取消按钮
- 背景内容应被 `aria-hidden` 屏蔽

### BottomSheet
**[推断]**
- 焦点移入 Sheet，聚焦标题
- Radio Group 使用方向键切换选项
- "完成"按钮获焦提交
- 关闭时焦点返回触发元素

---

## 6. 减少动画（Reduce Motion）

**[推断]** 当系统设置"减少动画"时：
- Toast：直接渐显/渐隐，无位移
- Modal：直接渐显/渐隐，无缩放弹性
- Sheet：直接渐显，无滑入位移
- Radio 切换：直接状态改变，无弹性动画
