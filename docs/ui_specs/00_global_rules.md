# 00 Global Rules — 全局执行总则

## 1. 总目标

- 以 `web/design/source` 为唯一视觉真相
- 以 `assets/backgrounds` 和 `assets/characters` 为唯一复杂视觉资源来源
- 以本目录文档为唯一 UI 工程规范来源

## 2. 三条硬规则

### 2.1 不重设计

- 不改页面结构
- 不改组件比例
- 不改图文主次
- 不改角色设定
- 不改设计稿中的情绪表达

### 2.2 不重画

- 只要资源目录中已有图片，必须直接引用
- 不得重新生成背景
- 不得重新生成礼盒
- 不得重新生成引导页插画
- 不得重新生成 Splash
- 不得重新生成人物头像

### 2.3 不自由发挥

- 未被规范允许的变化，一律禁止
- AI 只能实现，不能优化视觉
- 所有“建议值”在本轮增强后都视为“固定值”，除非本目录明确写成可选

## 3. 资产优先级

必须严格遵守：

`已有本地资源 > 基础代码组件实现 > AI 自由生成`

展开解释：

- 复杂背景、角色图、礼盒、引导页插画、Splash 图，优先级最高，必须直接使用本地资源
- Button、Dialog、Toast、BottomSheet、Input、TabBar、List、Typography、Bubble、Waveform 等基础界面元素，必须由 React 组件实现
- 不存在“允许 AI 再生成一张差不多的图”这种路径

## 4. 浅色/深色模式规则

- 若页面在 `01_asset_mapping.md` 中明确指定了浅色/深色背景图，则必须按指定图切换
- 若页面未指定复杂背景资源，则只能使用规范中定义的纯色或渐变 token
- 不能在未指定的页面擅自套用其他背景图

## 5. 偏差容忍度

- `390×844` 逻辑尺寸下，主布局偏差不得超过 `2pt`
- 颜色、透明度、阴影、光效、圆角、字阶只能做工程等价实现，不允许肉眼可见偏差
- 响应式只允许做同比缩放和安全区适配，不允许重新排版

## 6. 组件与资源边界

- 资源负责“复杂视觉内容”
- 组件负责“交互、结构、状态、动画、可访问性”
- 不允许把交互组件做成图片
- 不允许把复杂资产用 CSS 简化替代

## 7. 页面交付最低要求

每个业务页面都必须同时具备：

- 正确资源引用
- 正确布局
- 正确浅色/深色表现
- 正确状态机
- 正确转场
- 正确返回规则
- 正确安全区
- 正确响应式缩放
