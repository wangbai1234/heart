# 03 Layout Constraint — 布局约束

## 1. 基准设备

- 逻辑尺寸：`390×844`
- 设计画布：`1024×1536`
- Safe Area Top：`47pt`
- Safe Area Bottom：`34pt`

## 2. 布局基线

- 页面主内容左右基准 gutter：`16pt`
- 文本类 section 左右基准 gutter：`20pt`
- 卡片圆角主值：`24pt`
- 次级卡片圆角：`16pt`
- 小组件圆角：`12pt`
- 胶囊按钮 / 输入框：`9999px`

## 3. 间距刻度

只允许使用以下间距刻度：

- `4`
- `8`
- `12`
- `16`
- `20`
- `24`
- `32`
- `40`

若设计稿存在非整数观感，以最接近刻度实现，但不允许改变视觉节奏。

## 4. Z-index 分层

- `z-0`：背景图
- `z-10`：页面装饰层
- `z-20`：主内容
- `z-30`：固定 Header / TabBar / Composer
- `z-40`：Toast
- `z-50`：Dialog / BottomSheet / 全屏过渡

## 5. Grid 与区域规则

- 所有业务页面采用单列移动端布局
- 不允许改成双列、瀑布流或桌面式布局
- iPad / Web 大屏适配时，页面主体保持移动端视觉宽度并居中，不重新设计信息架构

## 6. Safe Area 规则

- 所有顶栏必须避开顶部安全区
- 所有底部输入栏、TabBar、主按钮必须避开底部安全区
- 不允许让 Home Indicator 与主 CTA、Composer、BottomSheet 发生重叠

## 7. Blur / Glass 规则

- Blur 只允许用于毛玻璃容器，不允许拿来掩盖布局问题
- Glass 背景必须服务于卡片、Header、Composer、Bubble，不允许整页泛滥使用

## 8. 阴影规则

- 阴影必须柔和、低对比、长半径
- 不允许硬边投影
- 不允许明显 Android Material 风格阴影

## 9. 布局禁止事项

- 不允许擅自增加新 section
- 不允许重排主 CTA 位置
- 不允许改变头像、标题、正文的层级关系
- 不允许为“更响应式”而改成完全不同版式
