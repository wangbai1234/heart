## 触摸目标尺寸（iOS HIG 标准：≥ 44×44 pt）

| 可点击元素 | 视觉尺寸（估算值） | 触摸区域（估算值） | 是否达标 |
|-----------|-----------------|-----------------|---------|
| 返回按钮"<" | 约 12×20 pt | 44×44 pt（扩展触摸区） | 达标 |
| 更多按钮"···" | 约 20×5 pt | 44×44 pt（扩展触摸区） | 达标 |
| 角色头像（可点击进角色详情） | 48×48 pt | 48×48 pt | 达标（刚好） |
| AI Voice Bubble 播放按钮 | 约 32×32 pt | 约 80×56 pt（整个 Voice Bubble 行可点击） | 达标 |
| 发送按钮 | 40×40 pt | 44×44 pt（建议扩展） | 建议扩展 |
| 附加功能按钮"+" | 约 36×36 pt | 44×44 pt（建议扩展） | 建议扩展 |
| AI Bubble（长按触发） | 可变 | 整个 Bubble 面积 | 达标（面积远大于 44×44pt） |
| User Bubble（长按触发） | 可变 | 整个 Bubble 面积 | 达标 |
| Composer 文字输入框 | 约 220×40 pt | 整个 Composer 区域可点击进入输入 | 达标 |

**存在问题：**
- 发送按钮（40×40 pt）略小于 44 pt 标准，建议扩展为 44×44 pt 的触摸区域（视觉保持 40pt，hitSlop 扩展）
- 附加功能"+"按钮同上，建议 hitSlop 扩展

## 颜色对比度（WCAG AA/AAA 评估）

### WCAG 标准参考
- AA 正文（<18pt）：对比度 ≥ 4.5:1
- AA 大字体（≥18pt 或 14pt 粗体）：对比度 ≥ 3:1
- AAA 正文：对比度 ≥ 7:1

### 各组合评估

| 文字/图标颜色 | 背景颜色 | 对比度（估算值） | WCAG AA | 备注 |
|-------------|---------|---------------|---------|------|
| #3A3A4A（AI Bubble 正文）| rgba(255,255,255,0.75) ≈ #FFDFE8（混合后估算） | 约 5.5:1（估算值） | 达标 | 毛玻璃需按实际渲染色计算 |
| #FFFFFF（User Bubble 正文）| #A7C7E7 | 约 3.8:1（估算值） | 部分达标（AA 大字体） | 16pt 正文略低，建议复核 |
| #FFFFFF（发送按钮图标）| #FFB7C5 | 约 2.2:1（估算值） | 不达标 | 图标非文字，参考 UI 组件标准（3:1）亦不达标，建议加深背景或描边 |
| #3A3A4A（角色名）| rgba(255,255,255,0.55) ≈ #FFF4F0（估算） | 约 8:1（估算值） | 达标（AAA） | 良好 |
| #888888（在线状态文字）| rgba(255,255,255,0.55)（估算） | 约 3.5:1（估算值） | 部分达标（13pt 小字体 AA 需 4.5:1） | 建议加深至 #666666 |
| #999999（日期分隔符）| 背景渐变（约 #FFEEE8，估算） | 约 3.2:1（估算值） | 不达标（12pt 小字需 4.5:1） | 建议加深至 #888888 或加大字号 |
| #AAAAAA（语音 Caption）| rgba(255,255,255,0.75)（估算） | 约 2.8:1（估算值） | 不达标 | 建议加深至 #888888 |
| #999999（时长"0:18"）| rgba(255,255,255,0.75)（估算） | 约 3.2:1（估算值） | 不达标（13pt 小字） | 建议加深 |
| #BBBBBB（Placeholder）| rgba(255,255,255,0.65)（估算） | 约 2.0:1（估算值） | 不达标（Placeholder 可接受豁免，但建议 ≥2.5:1） | Placeholder 按 WCAG 2.2 有豁免，但仍建议适当加深 |

**对比度问题总结：**
- 严重问题：发送按钮白色图标在粉色背景上对比度不足
- 次要问题：部分辅助文字（在线状态、时长、Caption、日期）在毛玻璃背景上对比度偏低
- 正文（AI Bubble）：基本达标，需在实际渲染环境再次复核

| 元素 | 字号 | 是否 ≥12pt | 行高 | 是否 ≥1.4 | 评估 |
|------|-----|-----------|------|----------|------|
| AI Bubble 正文 | 约 16pt | 达标 | 约 1.6 | 达标 | 良好 |

| 角色名"小屿" | 约 17pt | 达标 | 约 1.4 | 达标 | 良好 |

## ARIA 建议

> 适用于 React Native / Web 端实现时的 ARIA role 和 attribute 配置。

| 组件 | 推荐 role | 推荐 aria-label | 其他 aria-* 属性 |
|------|----------|----------------|----------------|
| 返回按钮 | `button` | `"返回上一页"` | — |
| 角色头像（若可点击） | `button` | `"查看小屿的详情"` | — |
| 更多按钮 | `button` | `"更多操作"` | `aria-haspopup="menu"` |
| AI Bubble 正文 | `article` 或 `listitem` | — | `aria-label="小屿说：{消息内容}"` |
| User Bubble 正文 | `listitem` | — | `aria-label="我说：{消息内容}"` |

| Typing Indicator | `status` 或 `log` | `"小屿正在输入..."` | `aria-live="polite"` |
| 消息列表 | `log` | `"聊天消息列表"` | `aria-live="polite"`，`aria-relevant="additions"` |
| 文字输入框 | `textbox` | `"消息输入框"` | `aria-placeholder="想和小屿说点什么..."` |
| 发送按钮 | `button` | `"发送消息"` | `aria-disabled="true"`（输入为空时） |
| 附加功能按钮 | `button` | `"添加附件"` | `aria-haspopup="dialog"` |
| 日期分隔符 | `separator` 或 `time` | — | `datetime="2024-01-01T09:41"` |

## 移动端操作友好度

### 单手操作分析（右手拇指为主）

基于 iPhone 14（390×844 pt）的拇指自然弯曲可达区域：

| 区域 | 拇指可达性 | 评估 |
|------|----------|------|
| Composer（底部，约 710-774 pt 纵向） | 极易到达 | 优秀 |
| 发送按钮（Composer 右端） | 易到达（右手拇指核心区） | 优秀 |
| 附加功能"+"（Composer 左端） | 右手拇指需要稍伸展 | 良好 |
| Header 返回按钮（左上，约 20 pt） | 右手拇指需大幅伸展或换手 | 较差，符合 iOS 惯例（返回手势更友好） |
| Header 更多按钮（右上，约 370 pt） | 右手拇指需大幅上伸 | 较差，建议提供下拉手势替代 |
| 消息列表（中部） | 拇指可自然滚动 | 良好 |
| Voice Bubble 播放按钮（中部偏左） | 右手拇指需稍伸展 | 良好 |

### 拇指区域分布
```
┌──────────────────────┐
│  ×  难以到达区（Header）×  │  ← Header 操作不友好，依赖右滑手势返回
├──────────────────────┤
│                      │
│  ○  中等可达区（消息列表）  │  ← 滚动无问题，点击特定 Bubble 需调整
│                      │
├──────────────────────┤
│  ✓  拇指黄金区（Composer）  │  ← 输入框、发送按钮均在此区
└──────────────────────┘
```

**建议：**
1. 支持 iOS 原生边缘右滑返回手势（补充 Header 返回按钮的可达性问题）
2. 更多按钮的功能考虑提供长按 Header 区域呼出（或下拉手势）作为替代
3. 消息操作（复制/转发）通过长按 Bubble 触发，拇指可达性良好
