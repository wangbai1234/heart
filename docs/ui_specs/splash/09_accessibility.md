# 09 无障碍规范 — Splash Screen（启动屏）

> 启动屏为全自动展示的过渡界面，无用户主动交互元素。无障碍要求相对简单，但仍需在视觉可及性、屏幕阅读器支持、动画安全性等方面满足基本规范。

---

## 无障碍适用性概览

| 无障碍维度 | 适用性 | 说明 |
|------------|--------|------|
| 点击区域（Touch Target）| 不适用 | 启动屏无可交互元素 |
| 颜色对比度 | 适用 | 文字需满足 WCAG AA 对比度 |
| 焦点状态（Focus）| 不适用 | 无键盘焦点元素 |
| 文本可读性 | 适用 | 字号、字重需满足可读标准 |
| ARIA 属性 | 适用（最小化）| 页面角色声明 |
| 键盘导航 | 不适用 | 启动屏无键盘操作 |
| 屏幕阅读器 | 适用（基本支持）| 品牌信息应被朗读 |
| 动画安全 | 适用 | 需支持 `prefers-reduced-motion` |
| 移动端操作 | 适用（基本）| 无触控交互，但需关注可访问性声明 |

## ARIA 属性建议

### 页面角色声明

| 元素 | 建议 ARIA 属性 | 说明 |
|------|---------------|------|
| Splash 容器 | `role="img"` 或 `role="presentation"` | 整个启动屏作为品牌图像 |
| Splash 容器 | `aria-label="yuoyuo 正在加载"` | 向屏幕阅读器声明页面状态 |
| GemIcon 图片 | `alt="yuoyuo 心形宝石图标"` | 若使用 `<img>` 标签 |
| GemIcon 图片（装饰用）| `alt=""` + `aria-hidden="true"` | 若图标为纯装饰，可隐藏 |
| AppWordmark | 无需特殊声明（HTML 文字节点）| 屏幕阅读器自动朗读 "yuoyuo" |
| AppTagline | 无需特殊声明（HTML 文字节点）| 屏幕阅读器自动朗读 "陪你聊聊吧" |
| BreathingDots | `aria-hidden="true"` | 纯视觉装饰，对屏幕阅读器隐藏 |
| CloudDecorations | `aria-hidden="true"` | 纯装饰元素 |

### 推荐 HTML 语义结构

```
<div role="img" aria-label="yuoyuo 正在加载，请稍候">
  <!-- 背景装饰（aria-hidden） -->
  <div aria-hidden="true">...</div>

  <!-- 核心内容区 -->
  <div>
    <img src="..." alt="yuoyuo 心形宝石图标" />
    <!-- 或：<img src="..." alt="" aria-hidden="true" /> 若 wordmark 已传达品牌信息 -->
    <span>yuoyuo</span>
    <span>陪你聊聊吧</span>
  </div>

  <!-- 加载指示器（aria-hidden，视觉动画） -->
  <div aria-hidden="true">...</div>
</div>
```

## 屏幕阅读器支持（Screen Reader）

### 朗读顺序建议

| 朗读顺序 | 内容 | 说明 |
|----------|------|------|
| 1 | "yuoyuo" | App 名称（页面首要信息）|
| 2 | "正在加载" 或页面 `aria-label` | 状态信息 |
| （跳过）| 图标、云朵、三点 | `aria-hidden="true"` 隐藏 |

### VoiceOver / TalkBack 兼容性

| 平台 | 工具 | 建议 |
|------|------|------|
| iOS | VoiceOver | 启动屏加载期间，VoiceOver 用户应听到「yuoyuo，正在加载」的朗读 |
| Android | TalkBack | 同上，通过 `contentDescription` 实现（Capacitor WebView 中通过 ARIA）|
| Web | NVDA / JAWS | 通过 `role="status"` 或 `aria-live="polite"` 声明加载状态 |

## 动画安全（Motion Safety）

### prefers-reduced-motion 响应

iOS / Android 系统均支持「减少动效」无障碍选项。应用应响应此设置：

| 动画 | 正常模式 | 减少动效模式 |
|------|----------|-------------|
| 宝石进入动画（scale + spring）| 启用 | 改为简单 opacity fade（无 scale 弹跳）|
| 宝石浮动（上下晃动）| 启用 | 禁用（静止）|
| 三点呼吸动画 | 启用 | 改为静态三点（或极小幅度 opacity 变化）|
| 页面淡入/淡出 | 启用 | 保留（opacity 过渡属于安全动画）|

**规则**：

```
@media (prefers-reduced-motion: reduce) {
  /* 禁用所有 transform 类动画 */
  /* 保留 opacity 类过渡 */
}
```

### 闪光/闪烁安全

- 三点呼吸点和宝石星光粒子的频率需低于 3Hz（< 3次/秒），避免触发光敏性癫痫（WCAG 2.3.1）
- 估算：三点呼吸周期 1200ms ≈ 0.83Hz，符合要求
- 宝石浮动周期 3000ms ≈ 0.33Hz，符合要求

## 移动端操作建议

| 场景 | 建议 |
|------|------|
| 启动屏显示期间用户误触屏幕 | 吞噬触摸事件，不产生任何反馈（`pointer-events: none` 或 Touch event 拦截）|
| 系统返回手势（Android）| 拦截返回手势，防止用户在启动屏阶段退出 App |
| 截图/录屏 | 不需要特殊处理（无敏感内容）|
| 强制竖屏 | 锁定竖屏方向，启动屏不支持横屏 |
| 深色模式（Dark Mode）| 启动屏设计稿为浅色专属，建议在深色模式下保持原始色彩（不做反色适配），或提供独立深色版 |

## 无障碍符合性总结

| WCAG 标准 | 条目 | 符合度 | 备注 |
|-----------|------|--------|------|
| 1.1.1 非文字内容 | 图像替代文字 | 需实现 | 宝石图标需 alt 或 aria-hidden |
| 1.4.3 对比度（最小）| 文字 ≥4.5:1 | 基本达标 | tagline 建议加深颜色 |
| 1.4.6 对比度（增强）| 文字 ≥7:1 | 未验证 | 建议精确测量 |
| 2.3.1 三次闪光 | 闪光频率 <3Hz | 符合 | 呼吸动画频率安全 |
| 2.4.2 页面标题 | 页面有描述性标题 | 通过 aria-label | 建议 role + aria-label |
| 3.1.1 页面语言 | `lang` 属性 | 建议设置 `lang="zh-CN"` | 在 Capacitor WebView 的 HTML 根元素 |
