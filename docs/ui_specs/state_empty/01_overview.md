## 状态类型

本设计资产为**双分区画面（Dual-Panel Composite）**，在单张画布上垂直拼接了两个独立的、功能互补的 UI 状态：

### 上半区：空聊天状态（Empty Chat State）
- 这是用户第一次与角色 悠悠（yuoyuo AI）打开对话时看到的初始页面
- 聊天记录区域完全为空，无任何气泡
- 通过视觉插画 + 引导文案 + 建议 Pill 提示用户发起对话

### 下半区：AI 正在输入/加载状态（Loading / Typing State）
- 用于 AI 响应尚未到达时，让用户感知系统仍在运行
- 以骨架屏（Shimmer Bubble）替代实际消息气泡
- Header 中角色名下方显示"..."打点动画，指示 AI 正在思考

---

## 触发条件

| 状态 | 触发条件 |
|------|---------|
| 空聊天状态 | 用户首次进入与某个角色的对话页，历史消息记录为零 |
| 加载/骨架状态 | 用户发送消息后，AI 响应尚未返回；或历史消息正在从服务器加载中 |

---

## 退出方式

| 状态 | 退出方式 |
|------|---------|
| 空聊天状态 | 用户点击建议 Pill 或在输入框输入文字发送，进入正常对话流 |
| 加载/骨架状态 | AI 响应数据到达，骨架动画消失，实际气泡以渐入（fade-in）方式呈现 |

## 设计理念

### 核心原则：消除空白的焦虑，建立温暖的第一印象

**空聊天状态**的设计目标是将"无内容"的空白时刻转化为充满期待感和安全感的邀请时刻。通过梦幻插画和温柔引导文案，让用户在打开应用的第一眼就感受到"有人在等你"的陪伴感。

**加载状态**的设计目标是消除等待焦虑。Shimmer 骨架屏维持页面结构感，避免突然的空白闪现；打点动画（...）在 Header 中指示角色正在思考，拟人化增强临场感。

### 视觉语言
- 60% WeChat 清晰度（布局结构、Header、气泡位置逻辑）
- 40% Genshin 温暖感（插画风格、渐变配色、光晕质感）
- 二次元治愈风格：柔和云朵、宝石心形、暖粉渐变

## 情绪传达（用户在此状态下的情绪引导）

### 空聊天状态
用户情绪轨迹：**陌生感 → 好奇 → 温暖 → 想要开口**

- 心形宝石插画传递"珍贵、用心"——让用户感到被重视
- 粉紫云朵渐变营造"梦境边缘"的安全感——降低社交压力
- 引导文"我们刚认识，先聊点什么吧？"用第一人称拉近距离，语气轻松不沉重
- 三个建议 Pill（今天心情如何？/ 陪我说话 / 给我讲个故事）提供零门槛开口选择，消除"不知道说什么"的困境

**设计结果**：用户不会感到"这是个冷冰冰的空页面"，而是感到"她在等我，我随时可以开始"。

### 加载/骨架状态
用户情绪轨迹：**发送后的期待 → 轻微不安（是否收到？） → 安心（有反馈）→ 等待**

- Header 中的"..."打点动画明确告知"她正在想"，消除"消息是否发出"的疑虑
- Shimmer 骨架屏维持对话页的视觉结构，避免内容坍塌感
- 骨架颜色与背景色系一致（玫瑰粉系），不引发视觉突兀
- 整体轻盈、不焦躁，符合治愈系产品的节奏感
