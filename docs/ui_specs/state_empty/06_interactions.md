# 06 Interactions — 加载 / 空状态

## 交互总览

本页面涉及两类主要交互：
1. **空聊天状态**的用户引导交互（Pill 点击）
2. **骨架加载状态**的系统反馈动画（Shimmer + 打点动画）

---

## 1. 页面进入动画（Empty State Enter）

### 触发条件
用户从角色列表/主页点击角色卡片，进入对话页，历史消息为零。

### 进入序列

| 步骤 | 元素 | 动画 | 时长 | 延迟 |
|------|------|------|------|------|
| 1 | 页面背景 | fade-in（opacity 0→1）| 200ms | 0ms |
| 2 | Header（含头像、角色名） | 向下滑入 + fade-in（translateY -10px → 0, opacity 0→1）| 250ms | 50ms |
| 3 | EmptyIllustration（云朵+心形）| 向上浮入 + fade-in（translateY +16px → 0, opacity 0→1）| 400ms | 100ms |
| 4 | GuideText | fade-in（opacity 0→1）| 300ms | 300ms |
| 5 | SuggestionPills（依次出现）| fade-in + 轻微向上（translateY +8px → 0）| 250ms each | 400ms / 500ms / 600ms |

**设计稿未明确定义进入动画，以上为推荐实现。**

### 缓动
- 步骤 1–4：`ease-out`（cubic-bezier(0.25, 0.46, 0.45, 0.94)）
- 步骤 5（Pill）：`spring`（cubic-bezier(0.34, 1.56, 0.64, 1.00)）

## 3. SuggestionPill — Tap（点击建议词）

### 触发条件
用户点击任意一个建议 Pill。

### 交互流程

```
用户 Tap → Pill 进入 Pressed 状态（视觉反馈）
         → 释放后 Pill 恢复
         → 输入框填入 Pill 文字
         → 自动触发消息发送
         → 空聊天状态消失（fade-out）
         → 进入骨架加载状态（fade-in）
```

### Pressed 状态视觉反馈

| 属性 | 变化 |
|------|------|
| Scale | 0.96（缩小 4%） |
| Background | 略深（约 darken 5%）|
| Shadow | 消失或减弱 |
| 动画时长 | 100ms（按下），150ms（释放） |
| 缓动 | ease-in（按下）/ spring（释放）|

### 状态切换动画

| 步骤 | 动画 | 时长 |
|------|------|------|
| 空状态区 fade-out | opacity 1→0 | 200ms |
| 加载状态区 fade-in | opacity 0→1 | 300ms |

---

## 4. Shimmer 骨架扫光动画（Skeleton Shimmer）

### 触发条件
进入骨架加载状态时自动开始，AI 响应到达时停止。

### 动画规格

| 属性 | 值 |
|------|-----|
| 动画类型 | 线性扫光（Linear Sweep）|
| 方向 | 从左到右（LTR）|
| 扫光层 | `linear-gradient(90deg, transparent 0%, rgba(255,255,255,0.60) 50%, transparent 100%)` |
| 初始位置 | translateX(-100%) |
| 终止位置 | translateX(+200%) |
| 时长 | 1400ms per cycle |
| 缓动 | linear |
| 循环 | infinite |
| 应用元素 | 所有 SkeletonMessageBar 和 SkeletonAvatarPlaceholder |
| 同步性 | 所有骨架元素 Shimmer 同步启动（相同 animation-delay: 0ms） |

---

## 5. 打点动画（Dot Typing Animation）— Header Loading Variant

### 触发条件
进入骨架加载状态时，Header 副标题文字切换为"···"打点动画。

### 动画规格

| 属性 | 值 |
|------|-----|
| 内容 | 三个圆点，依次出现 |
| 每点出现时间 | 200ms 间隔 |
| 动画方式 | opacity 0→1（顺序淡入）|
| 全部出现后 | 整组 opacity 1→0（250ms），再循环 |
| 总周期 | 约 900ms |
| 循环 | infinite |

### 视觉参考
- 状态 1（0ms）：`. · ·`（第一点亮，其余暗）
- 状态 2（200ms）：`. . ·`（前两点亮）
- 状态 3（400ms）：`. . .`（全亮）
- 状态 4（650ms）：`. · ·`（循环重置）

## 6. 骨架消失 → 真实内容显示（Skeleton Reveal）

### 触发条件
AI 响应数据到达，服务器返回消息内容。

### 动画流程

```
AI 响应到达
    │
    ▼
Shimmer 动画停止（立即）
    │
    ▼
SkeletonBubble(s) fade-out（opacity 1→0，200ms，ease-in）
    │
    ▼
真实消息气泡 fade-in（opacity 0→1，300ms，ease-out）
    │（可选）slide-in（translateY +12px → 0）
    ▼
Header 副标题切换回"在线 · 愿意倾听你的一切"（cross-fade 200ms）
```

### 时序

| 事件 | 时间偏移 |
|------|---------|
| 响应到达 | T+0ms |
| Shimmer 停止 | T+0ms |
| 骨架 fade-out 开始 | T+0ms |
| 真实气泡 fade-in 开始 | T+150ms（骨架尚未完全消失，轻微交叠）|
| Header 副标题恢复 | T+300ms |
