# 04 Visual Constraint — 视觉约束

## 1. 不允许修改的视觉要素

- 颜色
- 透明度
- 阴影强度
- 渐变方向
- 发光范围
- 描边粗细
- 字体层级
- 主视觉比例
- 头像裁切风格
- 玻璃质感强弱

## 2. 颜色约束

- 以页面现有 `03_design_tokens.md` 和设计图实色为准
- 不允许“为了对比度更好”改色
- 不允许“为了更高级”降低饱和度
- 不允许加入设计稿没有的新品牌色

## 3. 背景约束

- 已提供背景图的页面，必须直接引用背景图
- 未提供背景图的页面，只能使用设计稿中的纯色或既有 token
- 不允许将纯色页改为插画页
- 不允许将插画页改为纯色页

## 4. 角色约束

- 不允许修改人物外观
- 不允许改变人物发色、配饰、裁切、光效
- 不允许使用新生成的人物头像代替本地头像资源

## 5. 视觉比例约束

- 首页 Hero、角色页 Hero、登录页 Hero、兑换页礼盒、引导页插画，都必须保持原始构图比例
- 只能裁边，不能压缩变形

## 6. 深浅色规则

- 浅色模式优先匹配浅色设计稿
- 深色模式只允许替换为规范指定的深色背景和深色 token
- 不允许把深色模式做成另一套设计风格

## 7. 视觉禁止事项

- 不允许自动“优化”间距
- 不允许自动“优化”视觉平衡
- 不允许自动“优化”按钮大小
- 不允许自动“补齐”缺失装饰
- 不允许自动“统一”所有页面的背景表现
