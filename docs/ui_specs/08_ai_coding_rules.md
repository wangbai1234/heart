# 08 AI Coding Rules — 专门约束 AI Coding

## 1. 禁止事项

### 1.1 禁止重新设计

- 禁止改页面结构
- 禁止改视觉主次
- 禁止改布局节奏
- 禁止改 CTA 位置

### 1.2 禁止重新生成

- 禁止重新生成背景
- 禁止重新生成人物
- 禁止重新生成礼盒
- 禁止重新生成 Splash
- 禁止重新生成 Onboarding 插画

### 1.3 禁止修改视觉值

- 禁止修改颜色
- 禁止修改透明度
- 禁止修改阴影
- 禁止修改渐变
- 禁止修改发光
- 禁止修改字阶
- 禁止修改组件比例

### 1.4 禁止生成占位图

- 禁止用 AI 临时生成 placeholder avatar
- 禁止用 AI 临时生成 placeholder hero
- 禁止把“待补资源”理解成“可自由补图”

## 2. 必须执行

- 必须引用本地现有资源
- 必须根据本目录状态机实现页面 UI 状态
- 必须根据本目录流转规则实现路由
- 必须使用真实 React 组件实现交互元素
- 必须支持浅色和深色模式
- 必须支持 iPhone 级安全区

## 3. 允许的工程自由

- 文件拆分方式
- hooks 命名方式
- store 选型
- Tailwind class 组织方式
- 组件目录结构

## 4. 不允许的工程自由

- 通过改设计来“降低实现难度”
- 用单张大图代替组件树
- 用 CSS 自绘代替已有复杂资源
- 为了适配 Web 桌面而重做布局
- 为了性能把视觉资源替换成简化版

## 5. 缺失资源处理规则

- 若规范已指定本地资源，则只能使用该资源
- 若规范未指定复杂资源，则只能使用纯色 / 渐变 token
- 若运行时业务数据缺少头像：
  - 先用真实业务头像
  - 若业务未返回头像，必须使用“空白占位头像组件”
  - 空白占位头像必须是中性、纯组件实现的占位态，不允许使用角色资源顶替
  - 不允许生成新头像

空白占位头像规则：

- 使用纯组件实现，不使用 AI 图片
- 允许为浅色 / 深色模式分别调整底色，但不得带人格化特征
- 不允许出现头发、五官、服饰、发饰等角色信息
- 建议表现：圆形容器 + 低对比中性底 + 简单用户轮廓线，或纯空白玻璃圆形

## 6. 主题实现规则

- 主题切换只允许改变规范指定的背景图和 token
- 不允许切换主题时改变布局或组件尺寸

## 7. 验收前自检

AI Coding 在提交实现前必须自检：

- 是否逐页引用了正确资源
- 是否任何复杂视觉被 CSS 替代
- 是否任何人物被替换
- 是否任何页面动线被修改
- 是否任何颜色或间距被“优化”
