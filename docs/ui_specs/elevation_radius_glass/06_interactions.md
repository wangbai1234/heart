# 06 交互规范 — Shadow / Radius / Glass Sample

> 本资产为静态设计令牌参考图（Phase 1 Foundation），非交互式页面。
> 以下所有交互规范均标注来源：
> - **"设计稿已定义"** — PNG 中可直接观察到的视觉状态
> - **"设计稿未定义"** — PNG 未体现，基于设计系统规范补充建议

---

## 1. 进入动画

**设计稿未定义。**

建议（供研发参考）：
- 整个参考卡片从 opacity 0 → 1 + translateY(8px) → 0，duration 320ms，easing ease-out
- 三个 Section 依次错开 60ms 出现（stagger 效果）

---

## 2. 退出动画

**设计稿未定义。**

建议：opacity 1 → 0，duration 200ms，easing ease-in

---

## 3. Hover 状态

### 圆角示例方块 / 阴影示例方块

**设计稿未定义。**

建议（桌面端，若存在）：
- 轻微 scale(1.02)，duration 150ms
- 阴影从当前等级升一级（如 shadow.card → shadow.sheet）

### 玻璃卡片

**设计稿未定义。**

建议：白色遮罩透明度轻微降低（+0.05），增加通透感，duration 150ms

---

## 4. Pressed（按下）状态

**设计稿未定义。**

本资产为参考图，方块均为展示用途，无需交互响应。

若在实际 App 中 `GlassCard` 作为可点击组件：
- scale(0.97)，duration 80ms，easing ease-in
- 恢复 scale(1.00)，duration 120ms，easing ease-out

## 7. Loading 状态

**设计稿未定义。**

本资产为静态图，不涉及异步加载。

若插画背景图片未加载完成，建议：
- 占位底色使用 `color.surface` (#FFF8F3)
- Skeleton 动画：shimmer 效果，从左至右，duration 1.2s，循环

---

## 8. Error 状态

**设计稿未定义。**

若插画背景图片加载失败：
- 降级为纯色背景（天空蓝 #A7C7E7 ~ 奶油白渐变）
- 玻璃卡片正常显示（不依赖底图内容）

## 15. 玻璃效果动效（GlassCard 特殊规范）

**设计稿已定义静态视觉，动效未定义。**

静态确认值（PNG 直接读取）：
- `backdrop-filter: blur(16px)` — 已确认
- 4种透明度变体（35% / 55% / 75% / 着色）— 已确认

动态建议（设计稿未定义）：
- GlassCard 进场：blur 从 0 → 16px，duration 300ms，easing ease-out（配合 opacity 0→1）
- 这种效果模拟"玻璃缓缓变清晰"的沉浸感
