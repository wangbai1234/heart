## 2. 颜色对比度（Color Contrast）

### 主文字对比度检验

| 前景色 | 背景色 | 对比比值 | WCAG AA（4.5:1）| WCAG AAA（7:1）|
|--------|--------|---------|----------------|----------------|
| `#3A3A4A`（Ink）| `#FFFFFF`（白色卡片）| 约 10.8:1（估算值）| ✅ 通过 | ✅ 通过 |
| `#3A3A4A`（Ink）| `#FFF8F3`（奶油白）| 约 10.2:1（估算值）| ✅ 通过 | ✅ 通过 |
| `rgba(58,58,74,0.60)`（次级文字）| `#FFFFFF` | 约 4.6:1（估算值）| ✅ 通过（勉强）| ❌ 未通过 |
| `#FFB7C5`（品牌粉）| `#FFFFFF` | 约 1.9:1（估算值）| ❌ 未通过（纯装饰用）| ❌ 未通过 |

### 关键结论

- `#3A3A4A` 在白色/奶油白背景上对比度优秀，适合正文
- `rgba(58,58,74,0.60)` 次级文字在白色背景上勉强通过 AA，建议避免用于重要信息
- `#FFB7C5`（樱花粉）**不能用于文字**（对比度不足），仅用于装饰图标

### 玻璃效果文字对比度

| 场景 | 风险级别 | 建议 |
|------|---------|------|
| 文字在 玻璃 35%（白色35%）卡片上，背景为插画 | ⚠️ 高风险 | 避免在 glass-35 变体上直接放深色文字，背景插画明暗不可预测 |
| 文字在 玻璃 75%（白色75%）卡片上 | 低风险 | 75% 白色覆层足够遮挡背景，文字对比度基本可保证 |
| 文字在 玻璃 55%（标准）卡片上 | 中等风险 | 建议前景文字保持 `#3A3A4A`，避免浅色文字 |
| 文字在 玻璃着色 卡片上 | 中等风险 | 需根据实际色调计算对比度 |

## 3. 焦点状态（Focus State）

### 本参考图

> N/A — 静态文档，无可交互元素。

### Token 对 App 的影响

| 规范 | 要求 |
|------|------|
| 键盘焦点环颜色 | 建议使用 `color.primary` #FFB7C5 或 `color.accent` #C8B6FF |
| 焦点环宽度 | 最小 2px solid |
| 焦点环偏移 | `outline-offset: 2px`（避免与 border-radius 重叠） |
| 圆角与焦点环 | 焦点环应与元素圆角匹配（`outline` 不自动继承 `border-radius`，需 `border-radius + outline` 配合） |
| 玻璃卡片焦点 | 焦点环需足够对比度，避免被毛玻璃效果淡化 |

## 4. 文字可读性（Text Readability）

### 字号建议（基于本 Token 体系）

| 场景 | 最小字号 | 推荐字号 |
|------|---------|---------|
| 正文、标签 | 12px | 14px |
| 次级描述 | 10px | 12px |
| 标题 | — | 18–32px |
| 移动端正文 | 14px | 16px |

### 玻璃效果文字可读性

- 在 `glass-35` 变体上放置文字时，必须确保文字有足够的不透明度（禁止半透明文字叠玻璃）
- 在动漫插画背景上，推荐使用 `glass-75` 或以上变体承载正文文字
- 若必须在 `glass-35` 上显示文字，建议添加 `text-shadow: 0 1px 4px rgba(0,0,0,0.2)` 增强可读性

## 5. ARIA 建议

### 若本参考图以 Web 文档呈现

| 元素 | ARIA 建议 |
|------|----------|
| 页面标题 | `<h1>` 元素 |
| Section 标题（圆角/阴影/玻璃）| `<h2>` 元素 |
| 每个示例方块 | `<figure>` + `<figcaption>` |
| 圆角方块（4px）示例 | `<figure aria-label="圆角半径 4px 示例">` |
| 阴影方块示例 | `<figure aria-label="阴影等级：柔和，模糊 6px，不透明度 0.04">` |
| 玻璃卡片示例 | `<figure aria-label="玻璃效果 35%，白色覆层 35%，背景模糊 16px">` |
| 底部署名 | `<footer>` |
| ♥ 图标 | `aria-hidden="true"`（纯装饰，不传递语义信息） |

### 若 GlassCard 在 App 中作为可交互组件

| 场景 | ARIA 建议 |
|------|----------|
| 可点击卡片 | `role="button"` 或原生 `<button>`，`aria-label` 描述卡片内容 |
| 卡片内有内容 | 确保内容文字被屏幕阅读器访问（避免 `aria-hidden` 误用） |
| 加载状态 | `aria-busy="true"` |
| 展开/收起 | `aria-expanded="true/false"` |

## 7. 屏幕阅读器（Screen Reader）支持

### Token 本身的影响

- 圆角（`border-radius`）：对屏幕阅读器无影响，纯视觉
- 阴影（`box-shadow`）：对屏幕阅读器无影响，纯视觉
- 玻璃效果（`backdrop-filter`）：对屏幕阅读器无影响，纯视觉

### GlassCard 实现注意

- 毛玻璃效果不影响内容的 DOM 结构，屏幕阅读器可正常访问内容
- 若 GlassCard 作为覆层（overlay）出现，需管理焦点陷阱（`focus trap`）
- 背景内容被 GlassCard 遮挡时，考虑为被遮挡内容添加 `aria-hidden="true"`

## 8. 移动端操作建议

| 方面 | 建议 |
|------|------|
| 圆角与手指操作 | `radius.md`（16px）及以上圆角适合拇指区域操作，视觉柔和不割手 |
| 玻璃卡片点击 | 背景毛玻璃不影响点击响应，确保 z-index 层级正确 |
| 阴影与层级感 | `shadow.modal`（40px）阴影足够强调模态层级，用户无需依赖颜色判断层级 |
| 小圆角（4px/8px）元素 | 在移动端若用于按钮，建议增加 padding 确保点击区域 ≥ 44pt |
| 防抖 | GlassCard 的 `backdrop-filter` 在低性能设备可能影响帧率，建议提供降级方案（去除 blur，保留不透明遮罩）|

## 9. 视觉减弱运动（Reduced Motion）

> 本资产为静态图，N/A。
>
> 若 GlassCard 进场有 blur 动画，需响应 `prefers-reduced-motion: reduce`：
> - 禁用 blur 渐变动画
> - 保留 opacity 0→1（快速，150ms 以内）
> - 完全不动也是可接受的降级

## 10. 色盲友好性

| 颜色组合 | 色盲类型风险 | 建议 |
|---------|------------|------|
| 粉（#FFB7C5）vs 蓝（#A7C7E7）| 红绿色盲（Deuteranopia）可区分（不同明度）| ✅ 安全 |
| 粉（#FFB7C5）vs 白（#FFFFFF）| 任何类型 | ⚠️ 低对比，仅用于装饰 |
| 炭黑（#3A3A4A）vs 白 | 任何类型 | ✅ 高对比，安全 |
| 玻璃遮层信息传递 | — | 玻璃效果不应是**唯一**的信息传递手段，需辅以文字/图标 |
